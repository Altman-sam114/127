# 阶段 5：长期路线图

本文档定义 v0 测试板之后的长期演进方向。目标是避免早期原型把项目锁死在“纯六角格战棋”里，同时保留快速实现和可测试性。

核心判断：

- v0 可以使用六角格测试板验证玩法闭环。
- 长期规则底层建议演进为 `ProvinceNode / RegionNode` 省份图或区块图。
- 六角格可以继续作为早期显示方式或局部战术层表现，但不要作为全部大战略系统的唯一底层。
- LLM agent 只负责高层意图、作战计划和角色表达，不直接绕过规则系统修改状态。
- 前线、补给、组织度、撤退、补员和多级指挥链是长期核心系统。

---

## 1. v0：六角格测试板

### 版本主题

验证最小可玩闭环：地图、移动、攻击、占领、补给、MockAI、结构化命令执行。

### 核心新增系统

- 小地图，六角格显示。
- 德军 vs 盟军。
- 少量陆军单位。
- 移动、攻击、占领、基础补给。
- MockAI + 一个将领 agent。
- 验证结构化命令执行。

### 暂缓内容

- 不做完整经济。
- 不做空军、海军、天气。
- 不做真实本地 LLM 强依赖。
- 不做多级指挥链。
- 不做角色忠诚、抗命、政变。

### 技术重点

- `GameState`、`RuleEngine`、`Command`、`DecisionProvider` 必须从第一版开始解耦。
- 玩家和 AI 都只提交命令。
- 所有命令必须先校验再执行。

---

## 2. v0.1：组织度与撤退

### 版本主题

让战斗从“血条归零”升级为更接近大战略的战斗模型。

### 核心新增系统

- `strength` / `organization` 双数值。
- 撤退、补员、恢复组织度。
- 包围导致补给和组织度惩罚。
- 简化战斗日志。

### 规则方向

- `strength` 表示人力、装备和单位完整度。
- `organization` 表示战斗秩序、士气、疲劳和指挥效率。
- 战斗失败优先表现为组织度崩溃和撤退，不是直接全灭。
- 无撤退路线且被包围时，单位才可能投降或被歼灭。

### 暂缓内容

- 不细分弹药、燃料、维修。
- 不引入完整装备生产。
- 不做复杂战斗宽度。

---

## 3. v0.2：省份图底层

### 版本主题

把地图规则层从“显示格子”升级为“战略区块”，为前线、补给、占领区和大战略地图做准备。

### 核心新增系统

- 把地图规则层抽象为 `ProvinceNode / RegionNode`。
- 六角格变成一种显示方式。
- 区块控制权、核心区、占领区。
- 城市、工厂、资源挂到区块上。

### 省份图说明

省份图不是固定形状格子，而是由一组相邻区块组成的图结构：

```text
ProvinceNode
- id
- name
- owner
- controller
- terrain
- neighbors
- city
- infrastructure
- supplyValue
- factories
- resources
- coreOf
- occupationState
```

前线、补给、占领、工厂、资源和战争目标都应该绑定在 `ProvinceNode / RegionNode` 上，而不是直接绑定在 SpriteKit 渲染节点上。

### 技术重点

- 保持渲染层和规则层分离。
- `HexCoord` 可以继续用于 v0/v0.1，但长期不应出现在所有高层系统里。
- 新增地图适配层，把省份图映射到具体显示形态。

## 4. v0.3：前线系统

### 版本主题

实现接近《钢铁雄心4》的前线管理，让部队可以自动排成战线并减少缺口。

### 核心新增系统

- 自动识别前线。
- 集团军分配前线。
- 自动填补缺口。
- 攻势箭头 / 突破方向。
- 元帅 agent 制定战区计划，将领 agent 执行局部计划。

### 前线定义

前线是己方控制区块与敌方控制区块之间的边界集合：

```text
FrontLine = edges where province.controller == friendly
                 and neighbor.controller == enemy
```

### 自动布防原则

- 优先填补没有己方部队覆盖的前线缺口。
- 优先守城市、河流、要塞、山地和补给节点。
- 装甲部队不平均铺开，应集中在突破点。
- 组织度低或补给差的部队自动后撤或休整。
- 前线太长而部队不足时，系统给出战线过度拉长警告。

### 技术重点

- 前线自动排兵优先用确定性算法。
- LLM 只输出前线意图，例如“阿登方向集中装甲突破”，不逐个控制每个单位。
- 具体移动、换防、攻击目标由规则系统和自动规划器拆解。

## 5. v0.4：聊天命令与角色服从

### 版本主题

让玩家通过自然语言指挥将领，同时引入角色关系、服从和抗命的雏形。

### 核心新增系统

- 玩家通过聊天框命令将领。
- 将领根据性格、忠诚、风险偏好回应。
- 命令可能被质疑、拖延或抗命。
- 保留玩家直接微操能力，但影响将领满意度。

### 交互方向

玩家可以用聊天框下达高层命令：

```text
古德里安，三周内突破阿登，优先夺取巴斯托涅并切断盟军北翼补给。
```

将领 agent 输出：

- 对命令的理解。
- 建议作战计划。
- 风险评估。
- 需要的部队和补给条件。
- 结构化计划 JSON。

### 角色服从方向

- 忠诚度高：更倾向执行玩家命令。
- 风险偏好高：更愿意接受冒险突破。
- 性格强硬：更可能质疑低质量命令。
- 玩家频繁越级微操：可能降低将领满意度和执行效率。

### 技术重点

- 聊天内容必须转换成结构化 `PlayerDirective`。
- agent 的自然语言回复和实际命令 JSON 分离。
- 抗命不能直接破坏游戏规则，只能影响命令生成、延迟、执行意愿或政治后果。

## 6. v0.5：国家与部长 Agent

### 版本主题

从单一将领 agent 扩展到国家级和内阁级 agent，为大战略角色扮演打基础。

### 核心新增系统

- 统治者 agent。
- 军工部长、陆军部长、外交部长。
- 经济、科技、生产建议。
- 战略目标由高层 agent 下发。

### 多 Agent 层级

建议指挥链：

```text
国家统治者 agent
-> 元帅 / 战区司令 agent
-> 集团军将领 agent
-> 师级规则 AI / 自动规划器
```

部长系统：

```text
军工部长 agent：生产、工厂、装备建议
陆军部长 agent：编制、训练、将领任免建议
外交部长 agent：外交、阵营、停战建议
情报部长 agent：敌情、破译、风险提示
```

### 角色扮演方向

- 玩家可以扮演国家统治者，直接调度全局。
- 玩家也可以后续扮演将领，听从统治者调度、接受任免。
- 长期可加入不满、派系、抗命、政治危机、政变。

### 技术重点

- agent 之间传递的是摘要和结构化意图，不共享全量真实地图。
- 高层 agent 不直接移动单位。
- 多 agent 决策必须有统一仲裁层，避免互相覆盖命令。

## 7. v1.0：大战略原型

### 版本主题

形成可持续扩展的大战略原型，具备战役、征服、经济、科技、生产和多 agent 协作。

### 核心新增系统

- 战役模式和征服模式。
- 经济、工厂、资源、税收、人力。
- 科技和装备生产。
- 空军实体化。
- 简化海军。
- 天气。
- 多国家、多战区、多 agent 协作。

### 大战略系统方向

- 资源影响工厂产出和装备生产。
- 人力影响补员、扩军和长期战争能力。
- 核心区提供更高人力、税收和稳定产出。
- 占领区提供资源和工厂，但有抵抗、治安和效率惩罚。
- 科技解锁新坦克、飞机、火炮和作战 doctrine。
- 空军实体化为机场、航程、制空、支援和轰炸。
- 海军先做简化舰队和海域控制，不做复杂舰船细节。
- 天气影响移动、视野、空军效率和战斗修正。

### 商业化与美术方向

- 将领购买和收集可以作为长期商业化方向。
- 将领、元帅、部长、国家统治者都可以拥有立绘、背景、性格和技能。
- 部队图片、地图美术、城市图标和战报 UI 应逐步替换原型资源。

### 技术重点

- 数据驱动：国家、单位、科技、将领、地图、事件尽量使用 JSON 或可编辑数据表。
- 规则驱动：所有 agent 输出都必须进入同一套命令校验和执行管线。
- 性能驱动：本地 LLM 只在关键决策点调用，不参与每个单位每一步。

## 8. 性能与简化原则

本项目的核心特色是本地部署 LLM，但本地推理成本必须被严格控制。

### 必须遵守

- 不让每个单位每回合都调用 LLM。
- 不让 LLM 直接读取完整地图真相。
- 不让 LLM 逐个安排所有底层移动。
- 不让 LLM 绕过规则系统。
- 不把自然语言回复直接当作可执行命令。

### 推荐架构

```text
玩家 / Agent 意图
-> 战场摘要
-> 角色 prompt
-> 本地 LLM 输出计划 JSON
-> JSON 解析
-> 命令校验
-> 自动规划器拆解
-> RuleEngine 执行
-> 战报和日志摘要
```

### 决策分层

- 统治者 agent：战争目标、外交、政治路线、战略资源分配。
- 元帅 agent：战区目标、前线计划、攻势方向。
- 将领 agent：集团军行动、突破、撤退、局部调兵。
- 规则 AI：寻路、攻击目标排序、补给判断、换防、填线。

### 摘要分层

- 高层 agent 读取国家级摘要。
- 元帅 agent 读取战区级摘要。
- 将领 agent 读取局部战场摘要。
- 玩家可通过 UI 查看 agent 的摘要、计划和执行结果。

---

## 9. 版本依赖关系

推荐顺序：

```text
v0 六角格测试板
-> v0.1 组织度与撤退
-> v0.2 省份图底层
-> v0.3 前线系统
-> v0.4 聊天命令与角色服从
-> v0.5 国家与部长 agent
-> v1.0 大战略原型
```

关键依赖：

- 没有稳定 `Command` 管线，不要接复杂 LLM。
- 没有组织度和撤退，不要做复杂前线。
- 没有省份图底层，不要做 HOI4 式大战略地图。
- 没有前线系统，不要扩展多元帅、多战区。
- 没有摘要和仲裁层，不要让多个 agent 同时改同一批单位。

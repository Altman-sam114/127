# v0.8 初级经济、生产、城市、地形与补兵实现记录

目标分支：`codex/v0.8-economy-production`

分支审计说明：本轮早期尝试新建 v0.8 分支时，曾因 `.git` 写入权限受限失败；期间当前 checkout 先后观察到 `v0.4-generals-command-ui-resume`、`v1.0-ui-ai-playtest`、`v0.5-marshal-decision-chain` 等非 v0.8 分支名，且工作树已有多批其他版本未提交改动。最终已通过受控审批成功创建 `codex/v0.8-economy-production`，但创建后仍观察到外部 checkout 漂移，交付前需再次核对当前分支。本记录以当前工作树真实源码为准，合并前仍必须重新确认文件级冲突、重复文件引用、public API 和文档口径冲突。

本记录说明 v0.8 当前落地的代码与算法。v0.8 目标是建立回合制初级经济系统，让城市、工厂、基础设施、补给和地形从“展示属性”进入回合结算、生产队列、自动补员和战斗/移动修正。执行入口仍必须收口到 `Command -> RuleEngine -> CommandValidator -> CommandExecutor`，不得让 UI、AI 或经济系统直接绕过规则系统修改 `GameState`。

## 1. 本分支完成内容

### 1.1 经济状态模型

新增 `WWIIHexV0/Core/EconomyState.swift`：

- `EconomyResources`
  - 三项资源：`manpower`、`industry`、`supplies`。
  - 构造和增减都钳制为非负值。
  - 提供 `canAfford`、`add`、`subtract`，作为生产、补给维护费和自动补员的统一资源运算。

- `CityLevel`
  - `none`、`village`、`town`、`metropolis`。
  - 目前由 region 城市、胜利点、工厂数、补给值和 hex city/fortress 推导。
  - 提供工业值与人力增长值。

- `ProductionKind`
  - `infantryDivision`
  - `panzerDivision`
  - `motorizedDivision`
  - `artilleryDivision`
  - `supplyStockpile`
  - 每种生产项内置资源成本、建造回合数和补给产出。

- `ProductionOrder`
  - 记录订单 id、阵营、类型、剩余回合、总回合、创建回合和可选部署 region。

- `FactionEconomyLedger`
  - 每个阵营一份总账。
  - 字段包括库存、上一回合收入、上一回合维护费、上一回合补员消耗、生产队列和最后更新回合。

- `EconomyState`
  - `ledgers: [Faction: FactionEconomyLedger]`。
  - 兼容旧存档：`GameState` 解码缺失 `economyState` 时回退 `.empty`。

### 1.3 经济规则系统

新增 `WWIIHexV0/Rules/EconomyRules.swift`。

#### 初始总账

`makeInitialState(map:factions:turn:)` 对每个阵营生成账本：

```text
stockpile.manpower = baseManpowerReserve + income.manpower * 2
stockpile.industry = baseIndustryReserve + income.industry
stockpile.supplies = baseSupplyReserve + income.supplies
```

当前基础储备：

- manpower：320
- industry：160
- supplies：180

若传入 factions 为空，则 fallback 到 `Faction.allCases`。

#### 城市等级推导

`cityLevel(for:map:)` 使用 region 和 hex 信息共同判断：

```text
如果 region.city、hex city/fortress、region.factories 都不存在:
  none
否则如果 capital、victoryPoints >= 5、factories >= 5:
  metropolis
否则如果 victoryPoints >= 2、factories >= 2、supplyValue >= 3:
  town
否则:
  village
```

该算法保留 Hex 权威：hex 上的 `baseTerrain == .city`、`cityName`、`fortressName` 会参与 city 判断；Region 只提供聚合经济字段。

#### 回合收入

`income(for:map:)` 遍历由该 faction 控制且 passable 的 region，并额外要求该 region 至少有一个真实己方控制 hex。这个守卫用于防止历史数据层把 `owner/controller == nil` fallback 成 `.allies` 时，中立或空控制区被经济系统错误计入收入。

```text
coreBonus = region.coreOf 为空或包含该 faction ? 1 : 0
manpower = max(1, cityLevel.manpowerGrowth + coreBonus * 4 + infrastructure)
industry = max(0, factories + cityLevel.industryValue + infrastructure / 3)
supplies = max(1, supplyValue * 3 + factories + infrastructure / 2)
```

如果 map 没有 region，则 fallback 到 controlled hex 数、城市 hex 数和 supply source 数，避免测试板无 region 时经济为空。

#### 生产排队

新命令：

```swift
Command.queueProduction(kind: ProductionKind)
```

校验：

```text
phaseAllowsCommands
active faction ledger.stockpile.canAfford(kind.cost)
```

执行：

```text
扣除 kind.cost
创建 ProductionOrder(id: order_<faction>_<kind>_<turn>_<queueIndex>)
追加到 active faction productionQueue
写 supply 日志
```

当前 UI 只允许玩家在 alliedPlayer phase、非 observer mode、active faction 为 playerFaction 时点按钮排产。规则层本身仍按 active faction 泛化，因此未来 AI 或聊天命令也能共用同一 `Command`。

#### 回合经济结算

`CommandExecutor.executeEndTurn` 当前顺序：

```text
SupplyRules.updateSupplyStates
EconomyRules.resolveFactionTurn(for: activeFaction)
SupplyRules.advanceRetreats
SupplyRules.applyEncirclementAttrition
VictoryRules.updateVictoryState
切换 activeFaction / phase / turn
resetActionsForActiveFaction
StrategicStateBootstrapper.refreshRuntimeState
appendEvent
```

`resolveFactionTurn` 对当前 active faction 做四件事：

1. 加本回合收入。
2. 支付全军战略补给维护费。
3. 若 supplies 不足，把该 faction 已 supplied 的单位降为 lowSupply。
4. 对安全后方单位自动补员。
5. 推进生产队列，并尝试部署完成的单位。

注意：v0.8 是初级回合经济。它按“结束当前 active faction 回合”结算，不是完整大战略日历制同时结算。

#### 补给维护费

`supplyUpkeep` 计算：

```text
每个未毁灭单位基础 2
装甲单位 +2
火炮单位 +1
```

可支付部分从库存扣除；短缺时记录 `lastUpkeep` 为需求值，并触发战略补给短缺事件。

#### 自动补兵补员

自动补员候选条件：

```text
division.faction == faction
!isDestroyed
!isRetreating
supplyState == .supplied
strength < maxStrength
不与敌军相邻
```

排序：

```text
缺失 strength 多者优先
缺失相同时 division.id 字典序优先
```

每个单位每回合最多恢复 2 strength。每点 strength 成本按兵种权重计算：

```text
manpower = max(4, round(8 + 6 * (1 - armorWeight)))
industry = max(1, round(1 + armorWeight * 5 + motorizedWeight * 2 + artilleryWeight * 3))
supplies = 1
```

补员通过 `Division.reinforceStrength` 修改 strength，并写 `.reinforce` 日志。它不会恢复 organization；本项目当前战斗核心仍是 strength、retreat、supply、encirclement。

#### 生产完成与部署

生产队列每次 faction 结算时让本阵营订单 `remainingTurns -= 1`。

完成规则：

- `supplyStockpile`：直接给库存加 `85 supplies`。
- 其他单位：调用 `deploymentHex` 找安全后方部署点。

部署点优先级：

1. 订单指定且符合部署资格的 region。
2. 当前控制且符合部署资格的 region，按经济/补给价值排序：

```text
cityLevel.industryValue * 3 + factories * 2 + supplyValue + infrastructure
```

3. 当前 faction 的 supply source hex。

region 部署资格是硬门槛，不只是排序偏好。必须满足以下任一条件：

```text
capital
cityLevel == town/metropolis
factories >= 2
infrastructure >= 4
supplyValue >= 3
```

候选 hex 必须：

```text
passable
controller == faction
没有单位
不与敌军相邻
```

部署成功后追加新 `Division` 到 `state.divisions`；部署失败则订单保留在队列，等待后续回合重试。

### 1.4 地形战术加成

`BaseTerrain` 新增：

- `armorSlowdownCost`
- `supportsInfantryDefenseBonus`

`MovementRules`：

```text
进入目标 hex 时，如果 division.isArmor:
  path cost += toTile.baseTerrain.armorSlowdownCost
```

当前地形额外装甲成本：

- plain：0
- hill：1
- forest/city/fortress：1
- mountain：2

`CombatRules`：

```text
装甲攻击 plain defender tile: attack * 1.2
装甲攻击有 armorSlowdownCost 的 defender tile: attack * 0.9
步兵重单位防守 forest/city/fortress: defense * 1.3
```

这些修正都读取防守方所在 hex 或进入 hex 的 `baseTerrain`，没有把 region 地形作为战术权威。

## 5. 验收重点

后续 Agent C 或人工验收应重点看：

- `queueProduction` 是否只通过 `Command` 进入 `RuleEngine`。
- 旧存档/旧初始状态是否能通过 economy bootstrap 得到账本。
- 结束回合是否按 active faction 结算经济，并且不影响另一方未结束的回合。
- 自动补员是否只发生在 supplied、非撤退、非敌邻的后方单位。
- 新生产单位是否只能部署到本方控制、空置、非敌邻，且位于首都、城镇/大都会、工厂、高基建、高补给或 supply source 的后方 hex。
- 地形加成是否只读取 hex terrain，不把 region 变成战术权威。
- Xcode project 是否只有一组经济文件引用，没有重复 UUID 或 target membership 漏项。

## 6. 遗留风险

- 本轮最终已创建 `codex/v0.8-economy-production`，但分支创建发生在已有大量未提交跨版本改动之后，且过程中观察到外部 checkout 漂移，合并前必须重新审查当前分支、分支基点和文件级冲突。

- 只做了轻量 parse 和静态审查，未通过 Xcode 编译、运行时、UI 或回归测试证明。
- AI 目前不会主动做生产策略；规则层支持 active faction 排产，AI 经济部长属于后续版本。
- 城市等级当前为推导式算法，尚无独立数据编辑 UI。
- `RegionDataSet.toRegions()` 仍有历史 fallback：owner/controller 缺失时最终落到 `.allies`。v0.8 经济收入已加真实 hex 控制守卫，但数据层中立语义仍建议后续单独修。

## UI 美观化

关键文件：

- `WWIIHexV0/UI/PlatformStyles.swift`
- `WWIIHexV0/UI/RootGameView.swift`
- `WWIIHexV0/UI/AgentPanelView.swift`
- `WWIIHexV0/UI/EventLogView.swift`
- `WWIIHexV0/UI/EconomyPanelView.swift`
- `WWIIHexV0/UI/DiplomacyPanelView.swift`

改动逻辑：

- `PlatformStyles` 增加跨平台背景色、panel stroke 和 selection tint，统一 iOS / macOS 主游戏面板视觉。
- `RootGameView` 的 AI tab 把 `lastWarDirectiveRecords` 传给 `AgentPanelView`，让玩家能看到 AI 实际生成的 zone directive。
- `AgentPanelView` 显示每条 `WarDirectiveRecord` 的 zone、attack/defend、tactic、成功/拒绝命令数、目标 region 和 diagnostics。
- `EventLogView` 把最近 60 条日志包装成 `LogDisplayEntry`，每条只计算一次分类；新增 `diplomacy` 日志分类样式。
- `EconomyPanelView` 改用 `PlatformStyles.systemBackground`，避免 UIKit-only 色值污染 macOS 主游戏入口。
- `DiplomacyPanelView` 把过小的 `caption2` 升为 `caption`，提升小屏和 macOS 窗口内可读性。

## 轻量性能优化

关键文件：

- `WWIIHexV0/UI/EventLogView.swift`
- `WWIIHexV0/SpriteKit/BoardScene.swift`
- `WWIIHexV0/Commands/WarCommandExecutor.swift`

算法改动：

1. 事件日志展示模型：
   - 旧逻辑在 SwiftUI body 内创建 `recentEntries`，并对同一条 entry 多次调用 `category(for:)`。
   - 新逻辑生成 `[LogDisplayEntry]`，把 `entry` 和 `category` 成对保存，列表渲染时直接读取。
   - 结果：减少 body 重算里的字符串扫描和 switch 重复调用。

2. 单位绘制排序：
   - 旧逻辑在 `sorted` comparator 内反复调用 `adapter.unitDisplayHex(for:)`。
   - 新逻辑先把每个 division 映射为 `(division, displayHex)`，排序 comparator 只比较缓存坐标。
   - 部署图层下复用一个 `WarDeploymentManager` 实例计算单位 role，避免每个单位创建一次 manager。

3. AI 投入上限：
   - `AttackIntensity.infiltration` 现在会在没有显式 `maxCommittedUnits` 时生成默认投入上限。
   - `feint` / `guerrillaWarfare` 保留各自低投入默认值。
   - 结果：渗透、袭扰、佯攻不会默认全线压上，更适合初版试玩观察。

## MockAI / 元帅链完善

关键文件：

- `WWIIHexV0/Agents/ZoneCommanderAgent.swift`
- `WWIIHexV0/Commands/WarCommandExecutor.swift`
- `WWIIHexV0/UI/AgentPanelView.swift`

当前默认链路：

```text
MarshalAgent
  -> MarshalBattlefieldSummarizer
  -> SimulatedMarshalLLMClient
  -> TheaterDirectiveDecoder
  -> TheaterDirectiveCompiler
  -> DirectiveEnvelope / ZoneDirective
  -> WarCommandExecutor
  -> RuleEngine
  -> WarDirectiveRecord / AgentPanelView
```

行为要点：

- `SimulatedMarshalLLMClient` 已按兵力比、压力、补给警告、纵深单位和敌 zone 数选择 `blitzkrieg`、`spearhead`、`breakthrough`、`pincerMovement`、`feint`、`defenseInDepth`、`elasticDefense`、`lastStand` 等 tactic。
- `TheaterDirectiveCompiler` 把 `focusRegionId`、`convergenceRegionId`、`coordinatedZoneIds`、`maxCommittedUnits`、`exploitDepth` 编译到 `AttackParameters`。
- `WarCommandExecutor` 根据 tactic profile 选择前线/纵深单位、机动单位、炮兵优先、弱点 region、深目标和 fallback hold。
- 本分支新增 `AttackIntensity.infiltration` 的执行解释，补上 “JSON 里有强度字段但执行不分流” 的一处缺口。

## 初版游玩测试记录方式

本分支不执行重测试，也不启动 app。初版试玩建议人工在授权后按以下观察点记录：

1. 开局界面：HUD、图层切换、Info panel、Unit / Region / Log / Economy / Diplomacy / AI tab 是否可读。
2. 玩家回合：选中单位、移动、攻击、hold、allow retreat、resupply 是否仍显示清楚。
3. AI 回合：AI tab 是否显示 raw JSON、command results、zone directives、diagnostics。
4. 地图层：hex / province / initial / dynamic / front / deploy 图层切换后单位、前线、部署颜色是否可区分。
5. 性能体感：拖动地图、切换图层、打开日志面板是否有明显卡顿。
6. 规则边界：任何玩家或 AI 行动都必须能在 `CommandResultSummary` 或 `WarDirectiveRecord` 中追溯到规则系统。

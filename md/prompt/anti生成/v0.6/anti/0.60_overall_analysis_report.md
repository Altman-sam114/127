# v0.6 大版本：战局纵深展现、补给管线与“秋季黄色方案”真实大张地图推演

## 1. 0.6 版本定位与价值
经过 0.4 的双轨交互机制与 0.5 的 LLM 大脑接入，游戏的核心骨架“智能决策反馈系统”已经基本成型。在 0.6 版本中，我们将目光投向“**游戏深度与战场质感**”。
本阶段不再着重折腾抽象系统，而是实打实地落地一套**真实战役地图 (阿登地区)**、**补给与后勤命脉生命线**，以及**贯穿全程的战争统计档案记录仪**，为游戏从“Demo测试版”正式跃升到“可发行为 Beta 测试版”奠定物质基础。

## 2. 核心架构与挑战分析

### 2.1 真实阿登大地图推演 (大型战役如何切片？)
我们不能一上来就画整个欧洲，而是聚焦在战术和剧本冲突最强烈的“阿登森林突破”上。由于我们在 0.41 之前刚好确定了古德里安、里布、魏刚等将领名单，这**极其完美地契合了 1940 年法国战役的“镰刀闪击 (秋季黄色方案)”**。

*   **多大比较好？(地图规模)**
    *   **建议尺寸**：在 Hex 六角格下，建议约为 `60 x 80` 到 `100 x 120` 之间。在 SpriteKit 中这个规模结合视口缩放 (Camera Node)，既不显得空旷疲劳，又能容纳充足的包抄机动空间，且寻路/性能算法不会爆炸。
*   **国家阵营分布**：
    *   **轴心国**：德国（主攻）。
    *   **同盟国**：法国、比利时、英国（防守与被穿插）。
*   **需要绘制的核心战区 (Theaters) 与核心省份 (Provinces)**：
    *   **战区分布**：必须至少分为三个天然战区。即【北部平原战区（诱饵佯攻）】、【中部阿登密林战区（核心突击点）】、【南部马奇诺防线战区（静态对峙点）】。
    *   **著名地标与省份控制点**：列日要塞 (Liege)、色当 (Sedan, 法国心脏大门)、马斯河沿岸 (Meuse River)、卢森堡 (Luxembourg)、南锡 (Nancy)。
*   **登场部队与将领**：
    *   **德国方**：A 集团军群 (Bock / 博客) 集中装甲师在阿登穿插；C 集团军群 (Leeb / 里布) 在南部马奇诺对峙。主力前卫将领为 **古德里安** 统辖的快速装甲军，单位需要至少出现 `Panzer Division (装甲师)`、`Motorized (摩托化步兵)`。
    *   **盟军方**：法军第一集团军 (Weygand / 魏刚)、自由法国火种 (de Gaulle / 戴高乐 装甲预备队)、英国远征军预备队 (Montgomery / 蒙哥马利)。部队为 `Infantry (步兵)`、`Static Garrison (静态要塞驻军)`、`Heavy Armor (重装甲师但无燃油)`。

### 2.2 补给与后勤细化 (Logistics & Supply Systems)
战争打的就是后勤。在这以前兵可以乱跑，0.6 必须终结这一点。必须实现一条没有后勤就死亡的底线。
*   **机制核心：主输血管与神经末梢**
    *   **补给源 (Supply Depots)**：特定的工业都市或地图边缘判定为补给中心。
    *   **连通性判定 (Trace path)**：每回合初，对每个 `Division` 执行后勤测算算法。若它距离最近的补给源寻路穿越了非本联盟控制的 Hex，或者被切断，即陷入 `Out of Supply (OoS)` 状态。
*   **影响反馈轴线 (OoS Debuff Levels)**：
    *   **1 级断粮**：无法恢复并且逐渐丢失组织度 (Organization)。攻击力下降。
    *   **2 级断粮**：机械化部队失去移动能力。
    *   **3 级绝境**：超过三回合未接通，每回合开始自动扣除实际战斗力生命值 (Strength)，直至自行溃散灭亡。

### 2.3 历史沉淀与数据统计日志优化 (Statistics & Analytics Post-Game)
*   **当前痛点**：目前日志只是一串控制台文本。玩家无法直观感受“我打了多惨”以及供赛后复盘分析。而且大模型读取过去数据的途径单一。
*   **核心功能清单**：
    1.  **全局资源与伤亡账本 (Casualty Ledger)**：详细记录每一回合德法双方的 `Casualties (伤亡人员)` 和物资损耗。
    2.  **战役断代史 (Battle Log Replay View)**：用更美观的图表/报表 (类似统计学柱状图) 展示随着回合推移，“装甲第一师”在色当消灭了多少个团，获得了什么勋章与战区评级。
    3.  **JSON 落盘持久化**：所有的 `WarDirectiveRecord` 与战斗计算产生的战损记录，必须经过序列化成为一个独立的存档或数据库。未来大模型不仅能靠摘要推理这回合，还能根据前20个回合的结构化历史日志，写出一篇逼真华丽的《每日战地公报 / Field Bulletin》读给玩家听！

## 3. v0.6 落地细分步调建议 (Sub-version Breakdown)

为了将此磅礴的计划稳重落实落地，建议再次细分三等份任务：

### ✅ **v0.61 高速补给与包围网算法实装**
不搞花里胡哨，优先在代码层和图层面上跑通“补给衰减”。在底层网格实现后勤阻断与连通性图 (Graph) 算法。测试装甲师突进过深脱节后的组织度崩溃后果，并且 UI 层能对断粮单位亮出鲜红的“缺乏补给标记”。

### ✅ **v0.62 史诗级地理学：阿登战役剧本与编辑器大修**
大规模投入 Tiled 地图 JSON 的配置工作。利用我们前面的编辑器设计，去搭建这个 60x100 的雄伟古战场。在不同 Region 上钉死色当、阿登、要塞、河流。并把德国 A 集团军群和法国的重型要塞军作为默认剧本 `Scenario1940` 对象写死在游戏启动项中，测试全战区大混战的性能！

### ✅ **v0.63 战地记者的打字机：UI 统计面板与记录持久层**
完成数据持久层的对接。建立一个新的 `DashboardView / ReportView`。当一局游戏结束，或是玩家按下高层会议按钮时，弹出绚丽的统计面板。同时将这个面板打下的战死者名单与战役历史导出一个高度整洁的 JSON 文件，尝试交还给你的（v0.5大模型）让它充当一次历史小说家，输出这段战局的高层总结！

## 1. 本分支目标

v0.7 的目标不是另开一套规则系统，而是在现有战争命令主链路中扩展战术表达和执行算法：

```text
MarshalAgent / ZoneCommanderAgent / 手写 ZoneDirective
  -> ZoneDirective / TheaterDirective
  -> WarCommandExecutor
  -> Command(move / attack / hold / allowRetreat)
  -> RuleEngine / CommandValidator
  -> StrategicStateSynchronizer
```

核心边界：

- Hex 仍是战术权威；推进只改成功移动到的具体 hex。
- Region / Theater / Front / Deploy 仍是从 hex 与单位位置派生。
- AI、玩家、模拟 LLM 都只能产生 `ZoneDirective` 或上游 `TheaterDirective`，不得直接改 `GameState`。
- 不新增“直接造成伤害”的战术命令；所有战术都降级成现有底层 `Command`。
- 不跑 Xcode / XCTest / 模拟器 / Probe / Smoke / Full。

## 2. 并发子 Agent 使用

本轮使用两个只读子 agent：

- 子 agent 1：只调查 `WarDirective.swift` / `WarCommandExecutor.swift` 的扩展点，未写文件。
- 子 agent 2：只调查 `ZoneCommanderAgent.swift` / `MockAICommander.swift` / `TurnManager.swift` 和相关测试语义，未写文件。

## 3. 命令 schema 扩展

### 3.1 TacticName

`TacticName` 从二元扩展为：

```text
offense:
  standardAttack
  blitzkrieg
  spearhead
  breakthrough
  pincerMovement
  fireCoverage
  feint
  guerrillaWarfare

defense:
  holdPosition
  elasticDefense
  defenseInDepth
  lastStand
```

### 3.2 AttackParameters

新增字段：

```text
focusRegionId: RegionId?
supportRegionIds: [RegionId]?
convergenceRegionId: RegionId?
coordinatedZoneIds: [FrontZoneId]?
maxCommittedUnits: Int?
exploitDepth: Int?
```

用途：

- `focusRegionId`：定点突破主目标。
- `supportRegionIds`：侧翼、炮兵掩护或辅助推进方向。
- `convergenceRegionId`：钳形攻势会师区域。
- `coordinatedZoneIds`：参与协同的 front zone。
- `maxCommittedUnits`：佯攻、游击、矛头投入上限。
- `exploitDepth`：允许把会师/纵深目标加入候选的深度意图，不直接越过移动规则。

旧 JSON 缺这些字段仍可解码，因为字段都是 optional。

### 3.3 DefenseParameters

新增字段：

```text
fallbackRegionIds: [RegionId]?
counterattackRegionIds: [RegionId]?
strongpointRegionIds: [RegionId]?
maxFrontCommitment: Int?
```

用途：

- `fallbackRegionIds`：弹性防御/纵深防御后撤或调整目标。
- `counterattackRegionIds`：纵深预备队反击关注区域。
- `strongpointRegionIds`：死守/坚守重点。
- `maxFrontCommitment`：纵深防御中限制一线投入，保留空间和预备队。

旧构造调用仍兼容，新增参数都有默认值。

### 3.4 TheaterDirective 扩展

`TheaterDirective` 增加：

```text
convergenceRegionId
coordinatedZoneIds
```

并实现自定义 decode，旧 v0.5 JSON 缺 `coordinatedZoneIds` 时按空数组处理。

`TheaterDirectiveDecoder` 新增校验：

- `convergenceRegionId` 必须存在于 map region。
- `coordinatedZoneIds` 必须存在于 `WarDeploymentState.frontZones`。
- tactic/category 必须一致。

## 4. 战术算法

### 4.1 闪电战 blitzkrieg

目标：用机动兵力集中突破，突破后允许继续选择高价值纵深目标。

选择条件：

```text
mobileFriendlyStrength > 0
mobileFriendlyStrength / friendlyStrength >= 0.35
adjustedRatio >= 1.65
```

执行器 profile：

```text
includeDepthUnits = true
mobileOnlyWhenAvailable = true
weakPointFocus = true
allowDeepTarget = true
holdNonCommittedFront = true
```

行为：

- 优先选装甲/摩托化/高 movement 单位。
- 把 depth 机动单位加入攻击集群。
- 非矛头前线单位执行 hold，模拟步兵固定战线。
- 目标 region 按弱点评分重排。

### 4.2 定点矛头 spearhead

目标：单一突破口集中，不一定达到完整闪电战条件。

选择条件：

```text
mobileFriendlyStrength > 0
mobileFriendlyStrength / friendlyStrength >= 0.25
adjustedRatio >= 1.35
visibleEnemyRegionCount > 0
```

执行器 profile 接近闪电战，但默认投入：

```text
zone.unitsDepth.count + max(1, zone.unitsFront.count / 2)
```

用于“装甲/摩托化带头，部分步兵跟进”的定点突破。

### 4.3 突破 breakthrough

目标：常规优势下打开缺口。

选择条件：

```text
adjustedRatio >= 1.35
visibleEnemyRegionCount > 0
```

执行器 profile：

```text
includeDepthUnits = true
weakPointFocus = true
allowDeepTarget = exploitDepth > 0
```

与闪电战区别：不强制只用机动部队，步兵也可参与。

### 4.4 钳形攻势 pincerMovement

目标：多个 zone 朝同一个 `convergenceRegionId` 推进，形成会师和包围条件。

数据层：

```text
AttackParameters.convergenceRegionId
AttackParameters.coordinatedZoneIds
TheaterDirective.convergenceRegionId
TheaterDirective.coordinatedZoneIds
```

执行边界：

- 单条 `ZoneDirective` 只执行本 zone 的单位命令。
- 不直接把另一个 zone 的单位拉来执行。
- 多 zone 会师由元帅层或人工下发多条带同一 convergence 的 directive。
- 包围效果不硬编码，交给动态战区、前线、补给和包围规则派生。

### 4.5 火力覆盖 fireCoverage

目标：炮兵/远程单位优先压制，不主动冒进。

选择条件：

```text
artillerySupportStrength > 0
adjustedRatio < attackThreshold + 0.25
```

执行器 profile：

```text
includeDepthUnits = true
artilleryFirst = true
attackOnly = true
```

行为：

- 先排序炮兵/远程单位。
- 射程内有敌军则 attack。
- 没有目标时 hold，不移动进敌区。

### 4.6 佯攻 feint

目标：少量单位牵制敌军，避免全线投入。

选择条件：

```text
adjustedRatio < attackThreshold
并且 hasStaticDefense 或 hasContestedForwardPresence
```

执行器 profile：

```text
includeDepthUnits = false
committedUnitLimit = maxCommittedUnits 或 max(1, unitsFront / 3)
```

行为：

- 只投入少量前线单位。
- 能打则打，不能打则有限移动或 hold。

### 4.7 游击战 guerrillaWarfare

目标：用机动小群袭扰多个敌区/纵深高价值点。

选择条件：

```text
mobileFriendlyStrength > 0
visibleEnemyRegionCount >= 2
adjustedRatio >= 0.85
```

执行器 profile：

```text
includeDepthUnits = true
mobileOnlyWhenAvailable = true
weakPointFocus = true
allowDeepTarget = true
committedUnitLimit = 默认约 (unitsFront + unitsDepth) / 2
```

### 4.8 弹性防御 elasticDefense

目标：不在劣势下死扛，允许撤退避免被歼。

选择条件：

```text
pressure >= 2
或 supplyWarningCount > 0
或 adjustedRatio < 0.75
```

执行器行为：

- stance 强制 `.flexible`。
- 已在 front region 的单位执行 `.allowRetreat`。
- 不在 front region 的单位尝试补位，否则 `.allowRetreat`。

### 4.9 纵深防御 defenseInDepth

目标：一线迟滞，纵深预备队接应或反击。

选择条件：

```text
depthStrength > 0
并且 pressure >= 2 或 adjustedRatio < 1.0
```

执行器行为：

- 前线单位执行 `.allowRetreat`。
- 保留 `targetReserves` 个 depth 预备队。
- 其余 depth 机动单位若能看到 `counterattackRegionIds` 内敌军则 attack。
- 否则移动到 `fallbackRegionIds` / `strongpointRegionIds` 内防御加成较高、友控、可达的 hex。

### 4.10 死守 lastStand

目标：无纵深、极端劣势、压力高时全力坚守。

选择条件：

```text
adjustedRatio <= 0.35
depthStrength == 0
pressure >= 3
```

执行器行为：

- `targetReserves = 0`。
- 所有可行动单位按 holdLine 处理。
- 不主动撤退。

## 5. 定点突破评分

`WarCommandExecutor` 的 `bestBreakthroughRegion` 对候选 region 排序：

```text
enemyStrength 越低越优先
terrain.movementCost 越低越优先
有 road 越优先
city.victoryPoints + supplyValue + factories 越高越优先
guerrillaWarfare 额外计入 infrastructure
regionId 字典序兜底
```

候选来源：

```text
focusRegionId
commandTarget.region
convergenceRegionId
weightedRegions
supportRegionIds
front segment 相邻敌 region
```

若 `allowDeepTarget = false`，非邻接 region 不会进入候选；若为 true，只是允许选为移动方向，实际移动仍必须通过 `MovementRules` 和 `CommandValidator`。

# v0.7 大版本：战争艺术——战术体系大跃升与阿登闪击的实战验证

## 1. 0.7 版本定位与价值
如果我们前期的所有版本（0.3到0.6）都在解决游戏机制是否“跑得通”、“有没有补给”、“有没有经济”的硬件问题，那么 **0.7 就是要解决游戏“好不好玩”、“能否再现二战真实战术”的软件问题**。
目前系统里只有朴素的 `standardAttack (平推)` 和 `holdPosition (死守)`。但在 0.7 中，我们要深层改造 `WarCommandExecutor` 和 `RuleEngine`，引入真实的二战军事理论：闪电战、钳形攻势、纵深防御。让你的“阿登大图测试”真正展现出古德里安用装甲集群撕裂法军防线的史诗感。

---

## 2. 核心挑战：当前系统的战术瓶颈
在目前查阅的 `WarCommandExecutor` 中：
- `executeAttack` 的逻辑仅仅是“找到相邻敌军 Region -> 如果看见敌人就打 -> 如果没看见就往目的地走”。
- `executeDefense` 的逻辑仅仅是“找到防线薄弱点 -> 部队移动过去 -> 站桩防御 (hold)”。
这种算法只能打一战式的**“排队枪毙与战壕平推 (Attrition Warfare)”**，装甲部队和步兵没有区别，所有的兵力被迫均匀摊平在防线上。

## 3. v0.7 战术理论重构与算法落地方案

要在战棋中实现高级战术，我们必须升级 `TacticName`、增设专属指令解析，并让指令影响寻路权重与目标选择。

### 3.1 闪电战与定点突破 (Blitzkrieg & Spearhead Breakthrough)
- **理论解释**：不再把兵力平均摊在所有接壤战区。将所有高机动、高攻击的**装甲师 (Armor)** 和摩托化部队堆积在 1-2 个突破口 (Region)，不顾侧翼危险，强行凿穿防线深入敌后。
- **数据结构扩展**：
  在 `TacticName` 新增 `.blitzkrieg` 和 `.spearhead`。
- **算法落地**：
  当执行 `.spearhead` 时，`WarCommandExecutor` 会通过 `targetRegions` 抓取突破口。
  1. **部队分流**：步兵留在原地保持 `.holdPosition`。所有的装甲师强制收令向突破口 `targetRegion` 集中。
  2. **穿刺寻路**：一旦撕开防线进入突破口，装甲部队的寻路算法改为**“无视非阻挡性敌军，向敌方大纵深的高价值目标（如补给中心/城市）前进”**，从而实现穿插切断。

### 3.2 钳形攻势与包围网 (Pincer Movement & Encirclement)
- **理论解释**：两个相邻或相近的战区，朝着敌方突出部的后方同一点发起突击，最终在敌后汇合，将敌方大部队切断补给并闷死在口袋中。
- **数据结构扩展**：
  在 `AttackParameters` 中引入 `coordinatedTheaterId` 或 `convergenceRegionId`。
- **算法落地**：
  需要元帅层 (0.5) 的配合。元帅下达钳形攻势，指定 A 战区和 B 战区都向 C 区域（法莱斯口袋 / 基辅）突进。一旦 A 和 B 的部队在 C 区域胜利会师（占据了该 Region），包围圈形成，圈内敌军在 0.6 的补给算法下将迅速判定为 `Out of Supply` 而走向灭亡。

### 3.3 纵深防御与机动防御 (Defense in Depth & Mobile Defense)
- **理论解释**：不把部队堆在第一线给人炸。第一线只放少量兵力迟滞敌人，主力在后方预备队修筑阵地。一旦敌军装甲突入，主力利用空间换时间，在敌军疲惫时予以反击。
- **数据结构扩展**：
  完善 `DefenseParameters` 中的 `stance: .flexible`（当前已有占位符）。
- **算法落地**：
  1. **放弃死守**：当受到强力攻击（兵力比悬殊）时，`.flexible` 策略下的第一线部队不再执行 `hold` 然后被打死，而是抛出 `retreat (撤退)` 指令，主动向后方 `Depth Region` 移动。
  2. **机动反击**：当敌军主力进入由于我方撤退而空出的格子时（拉长了补给），一直按兵不动的预备队装甲师（在 `unitsDepth` 里的部队）将触发响应攻击，对其进行高额加成的侧翼包抄。

## 4. v0.7 落地细分步调建议 (Sub-version Breakdown)

为了将高级军事战术转化为可编程的条件判断代码，并结合阿登剧本（0.6）发挥威力，建议按如下次序实施：

### ✅ **v0.71 兵种特化与装甲集中 (The Armor Concentration)**
先对 `RuleEngine` 和 `WarCommandExecutor` 动刀。引入单位移动力的兵种识别（步兵、装甲）。实现“定点突破（Spearhead）” 逻辑：允许战区在下令进攻时，剥离装甲师独立集群，只向指定的一个最窄正面 Region 发起潮水般的攻击。在阿登地图上，测试古德里安的装甲军能否甩开步兵，单刀直入色当。

### ✅ **v0.72 撤退判定与弹性防御 (Flexible Defense)**
实装 `allowRetreat` 机制的核心代码（当前只是占位）。当防守战力小于敌军突击战力的 1/3 时，防守部队自动计算后撤路线并撤退，避免被瞬间全歼。随后实装“纵深防御 (Defense in Depth)”，让后方的预备队在撤退发生时接应。

### ✅ **v0.73 联合战役：大包围网 (Grand Encirclement)**
测试多战区协同作业。给两个相距很远的德国战区下达向同一个中心点目标的指令。在这个阶段，重点利用 0.6 的补给系统，去验证两支装甲矛头会师后，被切断的盟军是否真的一瞬间掉落到了灾难状态（无粮无弹大崩溃）。

### ✅ **v0.74 战术家大模型 (The Tactician LLM 进阶)**
此时系统已经具备了极度丰富的指令集（不光是攻防，还有大包抄、弹性防守、诱敌深入）。更新我们在 0.5 做的 LLM 解码器和兵法提示词。让元帅模型通过分析地图摘要，自己决定出：“盟军太厚，建议让伦德施泰特执行钳形攻势切断补给通道，曼施坦因实施机动防御”。让 AI 的军事才华真正震惊玩家！

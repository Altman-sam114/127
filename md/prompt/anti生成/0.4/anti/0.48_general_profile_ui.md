# v0.48 沉浸式将领信息面板 (类似世界征服者4 质感)

## 阶段目标
在 0.47 完善了指挥意图操作后，我们需要重点为将领系统补齐最后一块“角色扮演（RPG）”的拼图。本阶段要实现类似《世界征服者4》那样的高级将领信息 UI：玩家在点击战区/军队中的将领头像时，弹出精美、独立的将领档案面板，全面展示将领身世、技能体系以及当前服从度状态。

## 详细任务拆解

### 1. 将领档案弹窗面板开发 (GeneralProfileView)
- **触发入口**：在 0.43/0.47 制作的左下角/侧滑前线战区面板上，点击将领的“微缩头像”时触发。
- **弹窗形态**：在屏幕中央弹出带有毛玻璃背景遮罩 (`.ultraThinMaterial`) 的模态视图 (Modal / Popup Sheet)，阻断地图点击以聚焦当前将领信息。

### 2. 档案视图展示结构设定
整个面板需要划分为三个核心视觉区块：
1. **左侧 - 核心身份区 (Identity Block)**：
   - **大尺寸立绘 (Portrait)**：放置能够清晰展示将领半身像的高清素材框。
   - **姓名与军衔 (Name & Rank)**：在立绘下方用带有厚重历史感的排版显示将军全名及阵营国旗标。
2. **右上 - 履历与特质区 (Biography & Personality)**：
   - **个人简介 (Biography)**：一段简短的将军历史背景描述（拉取 0.41 中设定的 JSON `biography` 数据）。
   - **个性偏好槽位 (Personality Tags)**：用彩色标签展现这位将军的核心风格，如 `[红色盾牌: 风格激进 (Aggressive)]`、`[绿色徽章: 风险规避]`。
3. **右下 - 技能数据区 (Skills & Status)**：
   - **六边形或列表状的技能栏 (Skills Array)**：显示如“装甲精通”、“补给大师”、“防守大师”等具体战场加成天赋。
   - **动态关系面板 (Relationship Status)**：为之后的系统预留两个核心关系能量条 —— 【满意度 / 忠诚度】(Satisfaction / Loyalty)，利用 SwiftUI 的渐变进度条实现以体现玩家频繁干预带来的后果。

### 3. UI 交互动效与手感打磨
- 弹出与退出动画要干脆利落（推荐 `.spring(response: 0.3, dampingFraction: 0.7)` 级别偏生动的弹簧动画）。
- 技能图标支持长按弹出文字 Tooltip，介绍本技能的具体数值加成（如：防守时防守兵力系数 +15%）。

## 验收标准
- **面板呈现效果**：打开游戏加载任意包含古德里安或蒙哥马利的战区，点击小头像后，立刻感受到“打开了二战档案袋”的沉浸感。界面各处不显得空旷且信息层次符合现代移动端大作标准。
- **数据联通测试**：确保弹出的名字、简介、技能与 0.41 `GeneralRegistry` 解析到的深层设定完全匹配，并在不同战区间切换查看时做到数据正确刷新无缓存残留。

# v0.43 将领 UI 展示与指挥部逻辑 (General UI & HQ Logic)

## 阶段目标
在 0.41 和 0.42 中，我们已经实现了实体将领数据，并让他们通过指派器动态接管了战区。在 v0.43 中，我们需要将这些将领展示在 UI 上，使玩家能够直观地看到每个战区“是谁在指挥”，并解决**“是否需要实体指挥部单位”**的机制问题。

## 详细任务拆解

### 1. 指挥部逻辑敲定 (驻扎区块机制)
- **决断结论**：**不引入占格子的物理指挥部部队（HQ Unit）。**
- **理由**：引入实体棋子会导致严重的拥堵和寻路复杂化，且让原本纯粹的战区逻辑掺杂战术碰撞。
- **替代方案 (驻扎区块/HQ Region)**：
  - 在 `FrontZone` 或 `GeneralDispatcher` 发送指派时，为该将领在战区大后方（离前线最远或带有核心城市的 Region）挂载一个 `hqsRegionId` 属性。
  - **指挥瘫痪判定**：如果有敌方单位占领了该 `hqsRegionId`，则判定“指挥部被端”。将领可以在该回合强制触发“撤退重组”，并令所属战区该回合无法发动有效进攻（比如全额降级为防御指令）。不要求在这个阶段实现全部惩罚逻辑，但必须预留判定接口 `func isHQUnderAttack() -> Bool`。

### 2. 将领名片展示 (General Profile Panel)
- **目标组件**：在 `HUDView`、`AgentPanelView` 或者是类似 `RegionInspectorView` 里新增一处界面。
- **触发条件**：当玩家选中某个属于某战区 (`FrontZone`) 的区块（Region）或单位时。
- **UI 呈现要素**：
  - **头像 (Portrait)**：如果有配置图片则显示，无则用占位图或国家国旗垫底。
  - **将领名称**：如“古德里安 (Guderian)”。
  - **当前职位/战区**：如“阿登战区指挥官”。
  - **核心倾向与特质 (Tags/Badges)**：用小 Badge 显示 `CommandStyle` (如 Aggressive，用红色剑图标) 以及专长 Skills (如 Armor Expert)。
  - **状态条/数值预留**：预留出“忠诚度/满意度” (loyalty / satisfaction) 的槽位（UI 展示即可），这是为你计划中的 0.44 或之后玩家强行介入与违抗系统做可视化铺垫。

### 3. 指挥行为反馈日志扩展
- 现有的 `EventLogView` 应该可以明确反映哪位将军下达了命令，而不是冰冷的 generic AI。
- 例如更新 log 输出，使类似 `[German Commander] holding position` 变为 `[General Guderian] executes breakthrough via Bastogne`。

## 验收标准
- **UI交互表现**：当我用鼠标点选阿登的一个德军格子时，屏幕右侧或下方会优雅地滑出一个小面板，大字显示古德里安的头像及其好战标签。
- **数据串联**：确保 UI 不会发生硬编码错位，点击属于盟军战区的格子，能正确显示蒙哥马利或其他盟军将领。
- **机制验证**：战区内部存在 `hqsRegionId` 变量，可以通过外部改写状态模拟该 Region 易手，打印出“指挥部遇袭”的 debug 警告。

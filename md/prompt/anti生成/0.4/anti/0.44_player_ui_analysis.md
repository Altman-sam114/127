# v0.44 玩家半自动/全微操双轨指挥 UI 设计与系统分析报告

## 1. 核心挑战与概念辨析

在以往的二战兵棋游戏（如《钢铁雄心》、《欧陆风云》、《战争艺术》）中，玩家命令与 AI 自动化的边界往往是最大的技术痛点。
在 v0.44 阶段，我们要引入**全微操 (Micro)** 加上 **半自动/战役级宏观干预 (Semi-Auto)** 的双轨体系，这要求 UI 以及底层的命令解释器必须做到“**向下兼容且冲突自洽**”。

### 全微操模式 (Full Micro)
- **定义**：玩家扮演棋手，直接点击具体某个“师”，右键/拖拽到目标六角格，下达精确的移动、攻击、防守指令。
- **系统本质**：绕过 `ZoneCommanderAgent`（将领层）和 `ZoneDirective`（意图层），UI 交互直接产生基础的 `Command`（如 `MoveCommand`, `AttackCommand`），塞进 `RuleEngine` 并在回合结束时或者即时执行。

### 半自动模式 (Semi-Auto / General UI)
- **定义**：玩家扮演国家元首或战区大将。不直接管小兵，而是对整个前线、战区或者大兵团划定目标。
- **系统本质**：UI 交互产生的是高层 `ZoneDirective` 或新建立的 `PlayerDirective`。玩家相当于“手写了平时由 AI 自动生成的数据结构”。然后把这个指令下发给 `WarCommandExecutor` 让其调度部队执行，或者发给具体的将军让其评估是否接单。

## 2. 核心功能 UI 与机制实现方案

### 2.1 编制与部署 UI (部队角色划分)
**需求描述**：通过 UI 将部队分配为前线部署、预备队部署。
**底层映射**：对应着直接修改 `FrontZone` 模型中的 `unitsFront`（锋线部队）、`unitsDepth`（预备队/纵深部队）、`unitsGarrison`（驻防部队）。
**UI 实现方案**：
1.  **卡片组拖拽 (Drag & Drop)**：在屏幕底部的 `AgentPanelView` 弹出一整排该战区的兵牌。玩家可以将兵牌在【锋线槽】与【预备槽】之间拖拽。
2.  **快捷图层视图 (Overlay)**：在地图上按下 `Alt` 键或点击特定的“部署滤镜”，属于预备队的单位头上出现一个空心盾牌图标，属于锋线的单位是实心剑图标，玩家点击单位图标可快速循环切换其战区状态 (Front -> Reserve -> Garrison)。

### 2.2 宏观战术操作 UI (准备突破 / 集结 / 防御区)
**需求描述**：通过 UI 发布突破、集结、画防线等高级命令。
**底层映射**：产生 `ZoneDirective` (针对 `WarCommandExecutor`)，需要包含 `TacticName`、`category` 以及作用的 `regions`。
**UI 实现方案 - 画线与箭头引擎 (Planning Tools)**：
需要在 `SpriteKit` / `BoardSceneView` 引入一套**规划线绘制系统 (Planning Drawings)**。
1.  **画防线 (Defensive Line)**：玩家选中己方战区将军，点击“防御”，鼠标在几个连续的己方/边界 Hex 上划过。
    *   **转换器**：系统收集这些划过的 `Hex` 所属的 `RegionId`，生成一条 `DefenseParameters(stance: .holdLine, targetRegions: [选中区域])` 的 `ZoneDirective`。
2.  **准备突破 (Breakthrough Arrow)**：玩家右键拖拉（或者划线），拉出一个巨大的战略箭头，从己方战区指向敌方某个纵深 Region（如巴斯托涅）。
    *   **转换器**：捕捉箭头终点所在的 `targetTheaterId` 或 `targetRegion`。生成 `AttackParameters(targetTheaterId: xxx, weightedRegions: [箭头穿过的区域], intensity: .allOut)`。
3.  **视觉反馈**：这些玩家画好的图元（粗箭头、虚线圈），直接持久化存在 `FrontZone` 或专门的 `PlayerPlanningState` 里。只要指令没取消，这根战术箭头就在这（致敬钢铁雄心计划UI）。

### 2.3 微操命令 UI (单体部队操控)
**需求描述**：像传统战棋一样操控部队。
**UI 实现方案**：
这个目前最好做。点击某个 `Division`，高亮可移动区域 / 攻击目标。点击目标后，产生一条蓝色的微操连线。
微操下达的核心在于：**谁来拥有这支部队？**
如果玩家微操了一支属于古德里安战区的装甲师：
*   **强解绑方案**：系统自动将该单位从古德里安的 `FrontZone.assignedFrontUnitIds` 中剔除。这回合古德里安无法调度该师，它是“御林军”了。
*   **强覆盖方案**：单位还是战区的。但在 `RuleEngine` 收集命令时，玩家创建的 `Command` 优先级最高。将军派发的行动列表在过滤时，如果发现该部队已经被玩家微操指派，AI的指派直接抛弃。

## 3. 面向 0.44 的架构挑战与建议

在着手编写代码前，必须回答并发冲突问题。

### 挑战 1：AI 回合与玩家回合的时序冲突
*问题*：如果古德里安在回合初自己规划了攻击计划（生成了 AI Directive），玩家又在 UI 上划了进攻箭头（生成了 Player Directive），并且玩家还微操了其中三个师，谁说了算？
*架构建议（覆盖法）*：
1.  **最高优先级**：玩家针对单独 `Division` 的微操 `Command`。一旦存在，锁定该单位，AI 怎么排表都跳过它。
2.  **次高优先级**：玩家所画的 `PlayerDirective` 计划。如果玩家为某战区制定了强行突破路线，那么古德里安 AI 的评估决策阶段 (`makeDirective()`) 将直接被跳过，该战区强制使用玩家的 Directive 送去 `WarCommandExecutor` 拆解（此时可以加入 0.4 设想的将领可能发牢骚、忠诚度下降的判定）。
3.  **最低优先级**：没有任何玩家干预时，调用 `TheaterCommanderPool` 让将军按本性格自由发挥生成 AI Directive。

### 挑战 2：UI 画线手感的实现代价极大
*问题*：《钢铁雄心》的前线画图系统是上百人的团队迭代了好几代的产物。在 SpriteKit 中徒手用 `SKShapeNode` 连成平滑、不会产生穿模死角的“贝塞尔防线”和“进攻箭头”是非常耗费精力的。
*妥协性建议 (针对移动端及目前的 Hex 底层)*：
放弃自由手动画曲线！改为**“区域锚点选定法” (Region Anchoring)**：
- 玩家不要画线，而是**点击点**。
- `操作流：选择战区 -> 点击UI上的“攻势”按钮 -> 点击地图上的敌方 Region`。
- 地图自动在【玩家战区中心】和【点击的目标中心】之间连一条漂亮的虚线箭头。
这种“点击填空”交互逻辑远比自由画图实现难度低 80%，而游戏性完全一样。

## 4. 落地步骤建议（给 Agent 的行动路径）

为了实现 0.44 高难度的双轨制，应分三个阶段实施，严禁一步到位：
1.  **Phase A (微操底座)**：只做单单位全微操。玩家选中任何一个师，能够下达点对点 `Command`（并在渲染层显示红色/蓝色行动连线）。在执行 `RuleEngine` 时，过滤掉所有被玩家微操单位的 AI 指令。
2.  **Phase B (半自动指令层)**：不做画线 UI，只做文字面板。玩家点开战区，面板上有“全线防守”、“向 Region_X 突破”按钮，点击后生成强制的 `ZoneDirective` 注入指令列，观察下辖部队是否按照玩家指令运动。
3.  **Phase C (视图融合)**：在 SpriteKit 中补齐那些高大上的画线反馈与将领抗命反馈气泡。

这是当前能够最平稳落地的 0.44 双规制架构思路。

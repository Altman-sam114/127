# v0.45 玩家命令第二阶段：半自动宏观意图与战区改组

## 阶段目标
在完成了 0.44 单元微操不冲突的底座后，本阶段要实现真正的“将帅代理人”体验。玩家不需要一个一个师点，而是直接通过 UI 面板命令某个战区发起区域性行动（例如：要求阿登战区突破巴斯托涅），并且可以使用菜单调整哪一支部属于预备队。

## 详细任务拆解

### 1. 编制调整机制 (锋线与预备队改组)
- **需求**：在点击特定单位或者划定战区面板时，提供一个状态切换按钮（如 `Assign to Frontline` / `Assign to Reserve`）。
- **底层操作**：后台直接改写该战区 `FrontZone` 下的数组，将特定单位从 `unitsDepth` 移入 `unitsFront`，反之亦然。下一回合 AI 排兵布阵时会自动尊重新的编制配置。

### 2. 玩家高维意图拦截 (Player ZoneDirective)
- **UI 交互**：当玩家选中某位将军（战区面板）后，可以点击面板上的“部署攻势 (Offensive)”，然后点击地图上任意一个敌方控制的 `Region` 作为大目标。
- **系统解析**：后台捕获这次 UI 交互，组装成一条强效的 `ZoneDirective`。这会无视原定分配的 `ZoneCommanderAgent` 自己脑补的战局判断，强行生成 `AttackParameters(targetTheaterId: xxx, ...)` 塞入到这一回合的待执行管线中。

## 验收标准
- **部署验证**：测试通过点击，能让某个前线受损的师无缝划归为预备队，并在数据层观测到它从 `unitsFront` 转移到了 `unitsDepth`，随后在两回合内获得未交战的组织度恢复惩罚减免/红利。
- **半自动意图验证**：玩家在一个一直保持 `HoldPosition`（防守）的战区点选了“进攻 A 地”的按钮。回合末，原定防守指令被跳过，该战区所有可用锋线部队成功听从玩家指示向 A 地集火攻击。

# v0.47 玩家命令终极方案：UI 与命令层的全面统合及美化重构

## 阶段目标
经过 0.44 ~ 0.46 的洗礼，游戏底层的“单兵微操免疫体系”和“战区半自动发令体系”都已经成型。在 0.47 阶段，我们要进入“**集成与美学重构**”。
这个阶段的目标，是将零散挂载的半成品控制面板、调试级按钮，归拢并重新设计为一套**现代化、具备大作沉浸感、操作直观的扁平化指令 UI**，彻底将视图展示与命令下发引擎无缝结合。

## 详细任务拆解

### 1. 指挥中枢面板重构 (Command Center UI)
- **UI 版式架构**：抛弃简陋的 Debug Stack。请在底端或侧边引入使用现代设计（类似半透明 Glassmorphism 毛玻璃效果）的统一指挥中枢面板 (`CommandPanelView` 或类似统筹容器)。
- **层级切换响应**：
  - 点击“战区空白/统帅部” -> 显示前线数据、当前剩余战役指令点、全局将军后备池缩略图。
  - 点击“特定大将” -> 左侧滑出专属面板，拥有更精美的头像占位、显眼的忠诚条及性格 Badge，下方整齐排列“执行防御”、“筹备突击”等宏观战役指令大按钮。
  - 点击“单个师” -> 右下角滑出微操控制卡片，显示部队状态条与独立的 [攻击] / [撤退] / [移交储备队] 快捷操作区。

### 2. UI 与底层操作意图的无缝衔接 (Actions -> Directives/Commands)
- **事件流统一出口**：所有通过 UI 触发的行动意图，必须通过一个集中的 `UICommandDispatcher` / `PlayerActionHandler` 转化。
  - 当你在美化的“筹备突击”大按钮按下，地图提示你选择目标后，该事件流会封装出结构完全合法的 `ZoneDirective` 递交。
  - 当你在微操面板点击“向此移动”，立刻组装一条 `MoveCommand` 送入锁定序列。
- 清理以往堆积在 `GameView` 内部、由于前几个功能敏捷开发产生的“散装逻辑点”。令业务数据与 SwiftUI 视图解耦。

### 3. 美术特效与微交互动效 (Micro Animations)
- **过场反馈**：将军弹出抗命气泡或接受调遣时，引入 `.spring()` 等 SwiftUI 动画，令视图滑出显得柔和、具备重量感。
- **状态高亮**：被微操截断（变为豁免状态）的御林军师，在地图图层表现上给予特殊的金色/特殊颜色高亮圈底底盘，以便玩家可以一眼看出“这是我在微操的部队，那些是没有圈的是跟前指跑的部队”。
- **战术箭头的材质优化**：0.46 的草图箭头需要使用 SpriteKit 里更好的画刷特效材质和渐变色表现，比如代表德军突进则是从深灰色渐变为强烈的红/黑色箭头。

## 验收标准
- **外观呈现红线**：UI 彻底抛弃早期 Demo 质感，色彩协调，留白得当，字体有明确的主副层级。必须让开发者（和玩家）打开游戏时发出“wow，这 UI 像是个成熟产品”的赞叹。
- **操作闭环验证**：在经过高度美化封装后的侧边栏按钮和点选操作下，玩家依然能用不到 3 次点击，顺畅地下达“让古德里安强行冲锋（测试 45 大意图）”或“手动拉一个师回据点（测试 44 微操）”，并且操作如丝般顺滑，不引发系统冲突。

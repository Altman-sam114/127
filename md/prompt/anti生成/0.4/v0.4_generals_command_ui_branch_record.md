# v0.4 将军养成与玩家双轨命令实现分支说明

分支：`v0.4-generals-command-ui-final`

日期：2026-07-04

## 1. 本分支目标

本分支实现 `md/prompt/anti生成/0.4/anti/` 下 0.41-0.48 的轻量可运行版本：

- 将军实体数据：六位历史将军进入 `generals.json`。
- 将军分配：开局 region 种子和通用 dispatcher 把将军挂到 `FrontZone`。
- 将军养成初步：忠诚、满意度、玩家干预次数进入运行态。
- 将军 UI：侧栏面板和档案 sheet 展示将军、HQ、技能、辖下部队。
- 玩家微操：玩家可继续直接控制具体师，并锁定该师避免本回合将军指令覆盖。
- 玩家宏观命令：玩家通过将军面板下达防守/进攻 `ZoneDirective`。
- 地图反馈：宏观命令画计划箭头/防守圆环，微操单位画金色圈。

## 2. 非目标

- 不做真正抗命、政变、派系斗争或完整 RPG 成长树。
- 不接真实 LLM 或聊天自然语言解析。
- 不做自由手绘前线；采用 region 锚点自动连线。
- 不绕过 `RuleEngine`，不让 UI 或将军直接修改 hex / division。
- 不跑 Xcode / XCTest / 模拟器 / 全量构建。

## 4. 将军分配算法

入口：

```text
DataLoader.loadGameState
  -> WarDeploymentManager.makeInitialState
  -> assignGenerals(to:map:regionData:)
  -> GeneralDispatcher.assignGenerals
```

算法顺序：

1. 如果 FrontZone 已有合法同阵营将军，保留该将军，只刷新 `assignedDivisionIds`。
2. 读取 zone 下 region 的 `assignedGeneralId`，优先使用地图种子。
3. 如果没有种子，按 `preferredTheaterIds` / `preferredRegionIds` 匹配。
4. 如果仍无匹配，从同阵营未占用将军池取第一名。
5. 如果没有可用将军，`generalAssignment = nil`，保持安全空岗。

HQ 选择：

1. 优先选择 zone 内友方控制且有 city 的 region，按 `supplyValue` 高低排序。
2. 没有城市时，选择 display hex 数最多的 region。

HQ 遇袭判定：

```text
GeneralDispatcher.isHQUnderAttack
  -> assignment.hqRegionId
  -> map.region(id:)
  -> region.controller != zone.faction
```

## 5. 玩家微操锁算法

触发：

```text
AppContainer.submit(Command)
  -> RuleEngine.execute
  -> applyPlayerCommandBookkeeping
```

规则：

- 只在玩家 faction 的 `.alliedPlayer` phase 记录。
- 只记录有 `actingDivisionId` 的单位命令。
- 成功执行后把 division id 放入 `playerCommandState.micromanagedDivisionIds`。
- 该 division 所属 FrontZone 如果有将军，调用 `registeringPlayerIntervention(cost: 2)`，增加干预次数并降低满意度。
- `.endTurn`、active faction 改变或 turn 改变时清空本回合锁。

执行器过滤：

```text
WarCommandExecutor.execute(directive, in: state, excluding: lockedIds)
```

过滤点：

- 普通防御：front/depth 候选过滤 locked ids。
- 进攻：`attackingUnitIds` 过滤 locked ids。
- 进攻战术的非矛头 hold：remaining front ids 过滤 locked ids。
- 纵深防御：front/depth/counterattack 候选过滤 locked ids。

这样玩家本回合亲自控制过的师不会再被玩家宏观将军命令二次调度。

## 6. 玩家宏观将军命令算法

入口：

```text
GeneralCommandPanelView
  -> AppContainer.orderSelectedGeneralHoldLine
  -> AppContainer.orderSelectedGeneralAttackRegion
```

### 6.1 源战区推断

`selectedGeneralCommandZone` 按顺序推断：

1. 选中己方 division 时，用该 division 所在 `hexToFrontZone`。
2. 选中己方 region 时，用该 region 的 FrontZone。
3. 选中敌方 region 时，找与目标敌方 FrontZone 接壤的玩家 FrontZone。

### 6.2 防守命令

`Hold Line` 生成：

```text
ZoneDirective.defend(
  targetReserves: min(2, zone.unitsDepth.count),
  stance: .holdLine,
  tactic: .holdPosition
)
```

### 6.3 进攻命令

`Attack Region` 生成：

```text
ZoneDirective.attack(
  targetTheaterId: TheaterId(targetZone.id.rawValue),
  weightedRegions: [selectedEnemyRegion],
  intensity: .limitedCounter,
  focusRegionId: selectedEnemyRegion,
  maxCommittedUnits: min(3, zone.unitsFront + zone.unitsDepth)
)
```

执行方式：

```text
WarCommandExecutor(commandHandler: commandHandler)
  .execute(directive, in: startState, excluding: lockedIds)
```

这里不调用 `TurnManager.runDirectiveTurn`，因此玩家宏观命令不会自动结束回合。

## 7. 记录与 UI 反馈

执行后写入：

- `WarDirectiveRecord`
- `PlayerPlannedOperation`
- `lastWarDirectiveRecords`
- `lastCommandMessage`
- `interactionLog`

地图反馈：

- `BoardScene.drawPlannedOperations` 只读 `PlayerPlannedOperation`。
- attack：源 region/HQ 点到目标 region 点绘制箭头。
- defend：源点绘制防御圆环。
- `UnitNode` 对 `micromanagedDivisionIds` 内单位绘制金色圈。

UI：

- `GeneralCommandPanelView` 展示当前 FrontZone、将军、忠诚/满意度、干预次数、HQ 状态、辖下部队、目标 region、计划操作。
- `GeneralProfileView` sheet 展示履历、技能、关系条、HQ 状态和辖下单位。
- `RootGameView` 新增 `General` tab，Unit tab 内也嵌入将军面板。

## 目标

v0.9 在既有 hex / region / theater / front / deployment / marshal 指令链之上，新增一层国家统治者与初步外交状态：

- 多国家资料：在规则阵营之上保存国家身份，不扩写底层 `Faction`。
- 阵营集团：用 `DiplomaticBloc` 描述 Axis 与 Allied Coalition。
- 初步外交状态：保存国家间 `allied` / `atWar` 等关系。
- 统治者 agent：读取外交和前线摘要，对元帅/将军生成的 `DirectiveEnvelope` 做战略姿态塑形。
- UI 审计：新增外交面板，在 AI 面板展示最近统治者姿态。

- 不把 `Faction` 从 Germany / Allies 扩成多国家规则阵营。
- 不恢复旧 Cabinet / Minister / StrategicDirective 管线。
- 不让外交或统治者层直接修改 `GameState` 的 hex、单位、region、theater 或 front zone 权威状态。

## 数据模型

`DiplomacyState` 是 v0.9 的战略元数据根对象，挂在 `GameState.diplomacyState`：

```text
DiplomacyState
  countries: [CountryProfile]
  blocs: [DiplomaticBloc]
  relations: [DiplomaticRelation]
  rulerRecords: [RulerDecisionRecord]
  lastUpdatedTurn
```

当前开局种子：

- Germany 规则阵营：
  - `German Reich`
  - `Axis`
  - ruler agent `ruler_germany`
- Allies 规则阵营：
  - `United States`
  - `United Kingdom`
  - `Belgium`
  - `Allied Coalition`
  - 主 ruler agent `ruler_allies`

国家间关系按国家对生成。相同规则阵营为 `allied`，跨规则阵营为 `atWar`。`hostileCountryIds(to:)` 会对敌对国家做稳定去重，避免多盟友共享一个敌国时重复放大敌对国家数量。

## 统治者算法

`RulerAgent.adjust(envelope:in:)` 是唯一统治者入口。它接收已经由元帅/将军生成的 `DirectiveEnvelope`，返回调整后的 `DirectiveEnvelope` 与 `RulerDecisionRecord`。

输入摘要：

```text
GameState
  -> DiplomacyState.hostileCountryIds
  -> WarDeploymentState.frontZones
  -> FrontZone.pressure / frontSegments / unitsFront / unitsDepth
  -> Division strength / attack / defense
  -> WarDirectiveRecord defensive streak
```

派生快照：

```text
RulerStrategicSnapshot
  frontZoneCount
  averageZonePressure
  advantagedFrontZoneCount
  outnumberedFrontZoneCount
  contestedFriendlyPresenceCount
  hostileCountryCount
  staticDefenseStreak
  contestedRegionIds
  regionPriority
  zoneScores
```

姿态选择：

- `coalitionMaintenance`：敌对国家/联盟压力需要协调，且统治者 coalitionDiscipline 足够高。
- `defensive`：平均 zone pressure 高，或劣势 front zone 多于优势 front zone。
- `stabilizeFront`：近期连续防守，或前沿 contested presence 尚未稳定。
- `offensive`：统治者 aggression、riskTolerance 和优势 front zone 数合计足够高。

塑形规则：

- `offensive + attack`：将 `AttackIntensity` 提升为 `allOut`，并按 `regionPriority` 重排目标 region。
- `defensive + attack`：把攻击 directive 转成 `holdLine` 防御 directive，保留统一 `ZoneDirective` 后端。
- `coalitionMaintenance + defend`：提高 `targetReserves`，避免抽干联盟防线。
- `stabilizeFront + allOut attack`：降级为 `limitedCounter`。
- `stabilizeFront + defend`：改为 `flexible` 防御 stance。

统治者不执行命令；调整后的 `ZoneDirective` 仍由：

```text
WarCommandExecutor
  -> Command
  -> RuleEngine
  -> CommandExecutor
  -> StrategicStateSynchronizer
```

执行和同步。

## 回合链路

`.marshalDirective` 路径：

```text
TurnManager.runAITurn
  -> MarshalAgent.resolve
  -> TheaterDirectiveCompiler
  -> DirectiveEnvelope
  -> applyRuler
  -> diplomacyState.appendRulerRecord
  -> appendEvent(category: .diplomacy)
  -> executeDirectiveEnvelope
  -> WarCommandExecutor
  -> RuleEngine
  -> WarDirectiveRecord
  -> endTurn
```

`.zoneDirective` 显式路径同样经过 `applyRuler`，保证旧将军池 fallback 和手动 zone directive 后端也接受同一统治者塑形层。

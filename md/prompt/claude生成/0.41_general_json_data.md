# v0.41 实体将领 JSON 数据接入

## 任务总览

本任务属于 v0.4 将领系统第一步：将六位历史将领的实体数据写入 `general_agents.json`，并同步扩展 Swift 侧的 `GeneralAgentDefinition`、`ZoneCommanderAgentConfig`，以及 `TheaterCommanderPool`，使 `ZoneCommanderAgent` 能真正使用将领的个性属性驱动战术决策，而非依赖自动生成的占位配置。

**目标文件清单：**

| 文件 | 操作 |
|------|------|
| `WWIIHexV0/Data/general_agents.json` | 替换为六将完整数据 |
| `WWIIHexV0/Data/ScenarioDefinition.swift` | 扩展 `GeneralAgentDefinition` 结构 |
| `WWIIHexV0/Agents/ZoneCommanderAgent.swift` | 扩展 `ZoneCommanderAgentConfig`，强化 `TacticConditionChecker` |

## 步骤一：扩展 `GeneralAgentDefinition`

在 `ScenarioDefinition.swift` 的 `GeneralAgentDefinition` 结构中新增以下字段，所有新字段均用 `?` 或有默认值，保证旧 JSON 向后兼容：

```swift
struct GeneralAgentDefinition: Codable, Equatable {
    let id: String
    let name: String
    let faction: String
    let role: String
    let commandStyle: String          // 保留原字段
    let personalityPrompt: String
    let assignedDivisionIds: [String]

    // v0.41 新增
    let rank: String?                 // 军衔原文，如 "Field Marshal"
    let rankDisplay: String?          // 本地化显示，如 "元帅"
    let nationality: String?          // 可选，用于 UI 旗帜，如 "germany" / "france"
    let portrait: String?             // 肖像资源名，如 "portrait_montgomery"（预留，0.43 UI 使用）
    let biography: String?            // 简短传记，100 字以内
    let skills: [String]?             // 技能 tag 列表，对应 TacticConditionChecker 逻辑
    let aggression: Int?              // 0-100，进攻倾向，默认 50
    let riskTolerance: Int?           // 0-100，风险偏好，默认 50
    let preferredTactics: [String]?   // 偏好战术列表，["flanking", "armor_blitz"] 等
    let avoidTactics: [String]?       // 规避战术列表
}
```

**注意：** 不要删除 `assignedDivisionIds`，现有验证逻辑依赖它。`skills` 同名字段已存在于 `ZoneCommanderAgentConfig`，两者语义独立——JSON 侧的 `skills` 是静态描述，Config 侧的 `skills` 是运行时装载的版本。

## 步骤四：强化 `BinaryTacticClassifier`

当前 `styleBoost` 只有 ±0.15 三档，需用 `aggression` 字段做连续映射，并在计算中加入 `preferredTactics` / `avoidTactics` 判断。

修改 `BinaryTacticClassifier.classify` 方法，将 `config: ZoneCommanderAgentConfig` 接收到的 `aggression` 用于替代固定 `styleBoost`：

```swift
// 旧逻辑（替换掉）
let styleBoost: Double
switch config.commandStyle {
case .aggressive: styleBoost = 0.15
case .balanced:   styleBoost = 0
case .cautious:   styleBoost = -0.15
}

// 新逻辑：aggression 0-100 映射到 -0.3 ~ +0.3 的连续增益
let aggressionNorm = Double(config.aggression) / 100.0  // 0.0 ~ 1.0
let styleBoost = (aggressionNorm - 0.5) * 0.6            // -0.3 ~ +0.3
```

这样古德里安（aggression=85）的 boost = +0.21，蒙哥马利（aggression=35）的 boost = -0.09，比三档枚举更精细。

## 步骤五：激活 `TacticConditionChecker`

当前 `canUseTactic` 始终返回 `true`，现在加入 `avoidTactics` 检查：

```swift
struct TacticConditionChecker {
    func canUseTactic(
        _ tactic: TacticName,
        commander: ZoneCommanderAgentConfig?,
        zone: FrontZone,
        state: GameState
    ) -> Bool {
        // v0.41：如果战术在将领的回避列表中，返回 false（降级到 holdPosition）
        if let commander, commander.avoidTactics.contains(tactic.rawValue) {
            return false
        }
        return true
    }
}
```

**注意：** `TacticName` 需要有 `rawValue`。请先确认 `TacticName` 是否已为 `String` RawRepresentable。如果已有该特性，直接用 `tactic.rawValue`；如果没有，则在比较时用 `String(describing: tactic)`，或先给 `TacticName` 加 `: String, RawRepresentable`。

## 步骤六：新增六将 Swift 加载入口

在 `AgentConfiguration.swift` 中，仿照现有 `guderian` 加载模式（已存在但需更新），为六将统一提供从 `GeneralAgentDefinition` 构建 `ZoneCommanderAgentConfig` 的转换：

```swift
extension ZoneCommanderAgentConfig {
    /// 从 GeneralAgentDefinition（JSON 数据）构建 ZoneCommanderAgentConfig。
    /// assignedZoneId 由外部（指派器 v0.42）注入，此处用 placeholder。
    static func from(
        _ definition: GeneralAgentDefinition,
        assignedZoneId: FrontZoneId
    ) -> ZoneCommanderAgentConfig {
        let commandStyle: CommandStyle
        switch definition.commandStyle {
        case "aggressive": commandStyle = .aggressive
        case "cautious":   commandStyle = .cautious
        default:           commandStyle = .balanced
        }

        return ZoneCommanderAgentConfig(
            id: definition.id,
            name: definition.name,
            faction: Faction(rawValue: definition.faction) ?? .allies,
            assignedZoneId: assignedZoneId,
            skills: definition.skills ?? [],
            commandStyle: commandStyle,
            rank: definition.rank,
            rankDisplay: definition.rankDisplay,
            portrait: definition.portrait,
            biography: definition.biography,
            aggression: definition.aggression ?? 50,
            riskTolerance: definition.riskTolerance ?? 50,
            preferredTactics: definition.preferredTactics ?? [],
            avoidTactics: definition.avoidTactics ?? []
        )
    }
}
```

同时新增一个静态注册表（为 v0.42 指派器预备接口）：

```swift
struct GeneralRegistry {
    /// 从 DataLoader 加载所有将领定义，缓存为字典。
    static func load(from loader: DataLoader) -> [String: GeneralAgentDefinition] {
        let definitions = (try? loader.loadGeneralAgents()) ?? []
        return Dictionary(uniqueKeysWithValues: definitions.map { ($0.id, $0) })
    }
}
```

## 注意事项与边界说明

- **`assignedZoneId` 占位：** 本步骤不处理将领与战区的绑定关系，绑定逻辑属于 v0.42 指派器。`GeneralRegistry` 仅提供按 id 查定义的能力，`ZoneCommanderAgentConfig.from` 的 `assignedZoneId` 由外部传入。
- **`TacticName` rawValue：** 若 `TacticName` 没有 `String` rawValue，`avoidTactics` 比较需使用 `String(describing: tactic)` 或先扩展 `TacticName`。请阅读 `Commands/` 目录下的 `TacticName` 定义再决定写法。

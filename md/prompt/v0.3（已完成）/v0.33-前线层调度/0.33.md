# v0.33 战争部署与战区分层系统

## 执行摘要

v0.33 在 v0.31 Theater 与 v0.32 FrontLine 之上新增轻量战争部署层。核心目标是用 `FrontZone + FrontZoneSegment` 表达战区、前线切片和部队三层部署，不做 AI、不做寻路、不改 UI、不改 hex 战斗逻辑。

当前层级：

```text
HexCoord       显示 / 战术移动层
RegionId      规则层
FrontZoneSegment  前线调度切片
FrontZone      战区部署单元
WarState       战争强度状态
```

## 非目标

本版本不做：

- UI / SpriteKit。
- hex 逻辑重构。
- 前线 AI / LLM / agent。
- 路径搜索、最短路、全局优化。
- 复杂包围算法。
- 战斗规则改动。

## FrontZone 设计

`FrontZone` 是 v0.33 的独立战争部署单元。它由 v0.31 active Theater 初始化，但运行时使用自己的 `FrontZoneId`、region 映射、前线 segment 和单位池。

字段：

- `id`
- `name`
- `faction`
- `regionIds`
- `neighbors`
- `frontSegments`
- `unitsFront`
- `unitsDepth`
- `unitsGarrison`
- `pressure`
- `state`
- `isCoreZone`

规则：

- 一个 `RegionId` 只能归属一个 `FrontZone`。
- `FrontZone` 只读取 region 邻接，不扫描 hex。
- `neighbors` 由 region 邻接派生，包含友军和敌军相邻 zone。
- `pressure` 是 segment strength 的简化聚合，用于驱动 `WarState`。

## FrontZoneSegment 设计

v0.33 的最小调度单位是 region 粒度的 `FrontZoneSegment`。为避免和 v0.32 的 `FrontSegment` 类型冲突，代码中命名为 `FrontZoneSegment`。

字段：

- `id`
- `regionId`
- `neighborEnemyZone`
- `strength`
- `isEncircled`
- `assignedFrontUnitIds`

规则：

- 一个前线 region 最多生成一个 `FrontZoneSegment`。
- `neighborEnemyZone` 标记该 segment 面对的敌方 zone。
- 多敌战区时不拆多条全局线，对外仍由当前 `FrontZone.frontSegments` 统一暴露。
- `isEncircled` 使用局部邻接启发式，不做 flood fill 或路径搜索。

## 部队分层逻辑

部队按所在 region 和 zone 派生为三类：

- `FRONT_UNIT`：位于有前线 segment 的 zone，平均分配到 segment。
- `DEPTH_UNIT`：位于非核心、非前线接触的 zone，作为纵深预备队。
- `GARRISON_UNIT`：位于 core zone、城市或工厂 region，永不自动移动。

分配规则：

```text
if zone.isCoreZone or region has city/factory/core:
    GARRISON
else if zone.frontSegments is not empty:
    FRONT, assign to least-filled segment
else:
    DEPTH
```

当前版本只写入单位池，不移动单位坐标，不做路径规划。

## 战区演化规则

### 推进

入口：

```swift
advanceRegion(_:from:to:state:map:divisions:turn:)
```

流程：

1. 从 defeated zone 移除 region。
2. 加入 advancing zone。
3. 更新 `regionToFrontZone`。
4. 若 defeated zone 无 region，则删除该 zone。
5. 只重建推进 region 与邻区触达的 zone。

效果：

- FRONT 自动扩展到新的敌方边界。
- 旧边界若不再接敌，segment 自动消失。
- 被吞并 zone 删除后，相关 front 自动解除。

### 崩溃

入口：

```swift
collapseFront(zoneId:state:map:divisions:turn:)
```

规则：

- 当前 zone 处于 `totalWar` 或压力超过 region 数量时允许触发。
- 友军邻接 zone 被标记为高强度状态，作为 DEPTH 转 FRONT 的预留接口。
- 不移动单位，不做补线，不做 AI 决策。

### 消亡

当一个 zone 的所有 region 被吞并：

- `frontZones[zoneId]` 删除。
- `regionToFrontZone` 自动过滤失效映射。
- 邻接 zone 局部重建后，其 front segment 自动删除或转移。

## WarState

```swift
enum WarState {
    case peace
    case lowIntensity
    case highIntensity
    case totalWar
}
```

派生规则：

- 无 segment：`peace`
- pressure 低：`lowIntensity`
- pressure 中：`highIntensity`
- pressure 高：`totalWar`

v0.33 只计算状态，不根据状态执行复杂部署比例。

## NavalZone

`NavalZone` 作为海战接口保留：

- `id`
- `name`
- `linkedCoastalRegionIds`

当前不参与陆战部署、不参与 front segment 生成。

## 更新模式

### 初始构建

入口：

```swift
makeInitialState(map:theaterState:divisions:turn:)
```

从 v0.31 active Theater 建立 FrontZone，并对所有 zone 做一次 O(n) 初始派生。

### 事件更新

入口：

```swift
update(state:map:divisions:turn:events:)
```

事件类型：

- `regionControllerChanged`
- `frontZoneAssignmentChanged`
- `unitEntered`
- `unitLeft`
- `frontZoneChanged`

局部范围：

- dirty region 所属 zone。
- dirty region 的邻接 region 所属 zone。

不做全图扫描，不做路径搜索。

## 性能模型

| 操作 | 复杂度 | 说明 |
| --- | --- | --- |
| region neighbor lookup | O(1~6) | 六边格 / 省份邻接常数级 |
| 初始构建 | O(n) | n 为 active zone 内 region 总量 |
| event update | O(k) | k 为 dirty region 及其邻接 zone 的 region 数 |
| segment rebuild | O(n_front_zone) | 只扫相关 zone 的 region |
| unit assign | O(n_units_in_scope) | 只重置并分配相关 zone 单位 |
| encirclement check | O(local neighbors) | 只看敌接触 region 的一跳邻居 |

禁止路径：

- 不扫描 hex。
- 不跑最短路。
- 不做 graph clustering。
- 不做全局 AI 优化。

## 测试设计

### FrontLineTests

- 敌对 zone 接壤生成 front segment。
- 推进 region 后 front 自动扩展。
- 敌边界识别不依赖全局寻路。
- 事件更新只扫描 touched + neighbor zones。

### SegmentTests

- segment 使用 region 粒度。
- segment 正确记录 `neighborEnemyZone`。
- 局部包围结构可标记 `isEncircled`。

### DeploymentTests

- FRONT unit 平均分配到 segment。
- DEPTH unit 保留在纵深 zone。
- GARRISON unit 不移动。

### WarEvolutionTests

- 德军入侵法国：推进后 front 和 depth 邻接正确。
- 苏德战争：长战线多 segment 正确。
- 局部突破：包围 segment 标记正确。
- 战区消亡：zone 删除，front 自动解除。

## 测试模拟场景

### 场景 A：德军入侵法国

```text
berlin(core) - rhein(depth) - ardennes(front) - sedan(france) - paris(core)
```

验证：

- `ardennes` 生成德国 front segment。
- 德军推进 `sedan` 后，front 转移到 `sedan`。
- `rhein` 作为德国 depth 邻区保留。

### 场景 B：苏德战争

```text
g_north  - s_north
g_center - s_center - s_reserve
g_south  - s_south
```

验证：

- 一条长战线产生多个 segment。
- 多个 segment 仍归入同一 `FrontZone`。
- depth zone 不参与前线战斗。

### 场景 C：局部突破

```text
pincer_n
   |
pocket
   |
pincer_s
```

验证：

- 两个德国 segment 面对同一敌 zone。
- 敌方出口不足时标记 `isEncircled`。

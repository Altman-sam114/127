# v0.31-v0.33 总测试方案

## 目标

v0.34 真实地图改造前，只验证 v0.31-v0.33 战争链路是否可串联：

- hex -> region -> theater -> deployment
- theater -> frontline -> region segment -> deployment dispatch
- MockAI -> frontline/deployment snapshot -> structured orders -> command mapper -> rule engine

不把这批测试作为后续每次开发默认全跑项。

## 测试分层

### Smoke: 后续默认快速检查

只跑 `WarSystemsIntegrationTests`，共 4 个测试。

用途：

- 验证 DataLoader 生成 hex/region/theater/front/deployment 链路。
- 验证推进后 theater/frontline/deployment 同步变化。
- 验证 FRONT / DEPTH / GARRISON 分层部署。
- 验证 MockAI 可基于前线部署输出并执行命令。
- 验证无敌接触后退役战区不会继续暴露前线/部署段。

命令：

```sh
DEVELOPER_DIR=/Applications/Xcode.app/Contents/Developer xcodebuild test \
  -scheme WWIIHexV0 \
  -destination 'platform=iOS Simulator,name=iPhone 17 Pro' \
  -derivedDataPath /private/tmp/WWIIHexV0DerivedData \
  CODE_SIGNING_ALLOWED=NO \
  -only-testing:WWIIHexV0Tests/WarSystemsIntegrationTests
```

### Stage Regression: 阶段回归

只在改动 v0.31-v0.33 核心层时跑，当前共 45 个测试。

覆盖：

- `TheaterSystemTests`
- `FrontLineCreationTests`
- `FrontLineUpdateTests`
- `FrontLinePerformanceTests`
- `WarDeploymentFrontLineTests`
- `WarDeploymentSegmentTests`
- `WarDeploymentDeploymentTests`
- `WarEvolutionTests`
- `WarSystemsIntegrationTests`
- `AgentRegionCommandPipelineTests`
- `AgentPipelineTests`

触发条件：

- 修改 `TheaterSystem`
- 修改 `FrontLineManager`
- 修改 `WarDeploymentManager`
- 修改 `AgentContextBuilder` / `MockAIClient`
- 修改 region/theater/front/deployment 映射规则

### Full Test

不作为日常默认项。

只在以下情况运行：

- v0.34 真实地图接入完成后。
- 大范围重构规则引擎或数据加载。
- 合并阶段性里程碑前。

## 本次验证结果

Smoke：

- 4 tests passed

Stage Regression：

- 45 tests passed

## v0.34 后默认策略

v0.34 日常开发默认：

1. 跑受影响模块单测。
2. 跑 `WarSystemsIntegrationTests` 作为链路 smoke。
3. 不默认跑 45 个阶段回归。
4. 不默认跑 full test。

若 v0.34 改真实地图坐标、RegionGraph 或 theater/front/deployment 映射，再跑 Stage Regression。

## 性能说明

Mac 卡顿主要来自 iOS Simulator 测试链：

- Swift 编译
- 链接
- 安装 app 到模拟器
- 启动 XCTest
- 模拟器进程占用 CPU/内存/磁盘

测试本身执行时间很短；本次 45 个测试实际 XCTest 执行约 1 秒，长耗时主要在 Xcode/Simulator 启动与构建流程。

# v0.32 FrontLine System

## 执行摘要

v0.32 只实现战区级前线系统（FrontLine System）。前线定义为敌对战区之间的 Region 邻接边集合，不是 hex 路径，也不参与战斗结算。

当前层级：

```text
Hex（战术表现 / 移动）
  ↓
Region（控制 / 补给 / 规则）
  ↓
Theater（战略分区）
  ↓
FrontLine（战区边界结构，v0.32）
```

## 非目标

本版本不做：

- UI / SpriteKit 渲染
- AI agent / LLM 策略
- 命令系统
- RegionGraph 改动
- TheaterSystem 结构改动
- 完整 HOI4 前线
- 复杂战斗模拟

## 核心类型

### FrontLineId

`FrontLineId` 是 `RawRepresentable<String>`，用于稳定标识战区主前线。

### FrontLine

字段：

- `id`
- `theaterId`
- `opposingTheaterIds`
- `factionA`
- `factionB`
- `segments: [FrontSegment]`
- `type`
- `state`

规则：

- 一个战区面对多个敌战区时，对该战区只暴露一条主 `FrontLine`。
- 多个敌战区接触保存在 `opposingTheaterIds` 与 `segments`。
- 包围不拆分 FrontLine，只标记 `type = encirclement` 和 segment 状态。

### FrontSegment

字段：

- `regionA`
- `regionB`
- `edgeType`
- `pressureLevel`
- `supplyImpact`
- `isEncirclementCandidate`

`regionA` 属于当前战区，`regionB` 属于敌对战区。

### RegionFrontState

字段：

- `regionId`
- `frontLines`
- `lastUpdatedTurn`
- `dirtyFlag`

用于 region → frontline 查询。

### FrontLineState

字段：

- `frontLines`
- `regionStates`
- `enemyNeighborCache`
- `dirtyRegionIds`
- `lastUpdatedTurn`
- `diagnostics`

`diagnostics` 用于测试性能约束，如扫描 region 数和更新模式。

## 前线生成规则

前线 segment 生成条件：

1. `regionA` 属于当前 active Theater。
2. `regionB` 与 `regionA` 在 RegionGraph 邻接。
3. `regionB` 属于不同 Theater。
4. 两个 region controller 不同。
5. 两个 Theater 均非 inactive。

合并规则：

- 按 `theaterId + factionA vs factionB` 合并为一条主 FrontLine。
- 不跨第三方为独立 FrontLine；第三方敌战区作为 opposing theater 记录在同一主前线内。
- v0.32 不实现复杂连通组件拆分。

## 更新模式

### 模式 A：默认每回合重建

入口：

```swift
tick(state:map:theaterState:divisions:turn:mode:events:)
```

默认 `mode = .turnRebuild`，每回合最多一次。若 `lastUpdatedTurn == turn`，直接返回旧 state。

### 模式 B：事件驱动局部更新

入口：

```swift
update(state:map:theaterState:divisions:turn:events:)
markDirty(state:events:map:theaterState:)
```

事件类型：

- `regionControllerChanged(RegionId)`
- `theaterAssignmentChanged(RegionId)`
- `unitEntered(RegionId)`
- `unitLeft(RegionId)`
- `occupationChanged(RegionId)`
- `theaterChanged(TheaterId)`

局部更新范围：

- dirty region
- neighbor regions
- affected front segments

不做全图重算。

## 缓存

`enemyNeighborCache` 缓存：

```text
RegionId → [enemy neighbor RegionId]
```

用途：

- 避免重复 graph traversal。
- 支持 AI/UI 后续读取边界邻接信息。
- event-driven update 只刷新脏区和邻区缓存。

## 包围识别

v0.32 使用简化包围判断：

```text
friendlyContacts >= 2
and
(escapeRoutes < 2 or no local supply path)
```

标记效果：

- `FrontSegment.isEncirclementCandidate = true`
- `FrontSegment.pressureLevel >= 0.8`
- `FrontSegment.supplyImpact = high`
- `FrontLine.type = encirclement`
- 高压包围时 `FrontLine.state = collapsing`

## 战区扩展与删除

战区扩展：

- 外部系统改变 `regionToTheater` 后，发出 `theaterAssignmentChanged(regionId)`。
- FrontLineManager 局部更新对应 region 与邻区。
- 前线自动延伸到新边界。

战区解散：

- 外部系统将 Theater 标记为 `inactive` 后，发出 `theaterChanged(theaterId)`。
- 接触消失时 FrontLine 自动删除，相关 `RegionFrontState.frontLines` 清空。

## 性能约束

- 不扫描 hex。
- 不做 O(n²) region 对比。
- 每个 active Theater 只遍历自身 region 与邻接 region。
- event-driven update 只扫描 dirty region + neighbor regions。
- `FrontLineDiagnostics.scannedRegionCount` 用测试约束扫描范围。

# v0.351 初级战争命令系统与 MockAI 驱动编排 (Prompt for Codex)

> **前置任务说明**：本项目已经跑通了地图架构和编队战区底层（v0.33），并在 v0.34 实现了地图编辑器的生成打通。现在，我们的重点来到了大脑：**指令系统（Command System）**。
> 本次迭代要建立为后续真 LLM Agent 做铺垫的“初级版战争命令”架构，以及一个基于强逻辑计算起步的 MockAI。所有命令类型接口定义必须做到极度 JSON 友好，命名必须短促、清晰，以备未来 LLM 直接按这个结构做序列化输出。

请依据以下核心维度与要求，严密完成 Swift 代码编码与探针测试：

---

## 1. 面向 LLM 的“格式化指令协议体系”设计
不要把游戏移动攻击写死在 UI 层的触控函数里！我们需要在 `WWIIHexV0/Commands` 下全新设计一组基于 **“高级意图 (Directive)”** 驱动的结构。

### 要求设计的核心结构体 (必须高度支持 JSON 解析)
**通用封装 `DirectiveEnvelope`**：
为了方便未来大模型输出，请制定统一的根结构，例如：
```swift
struct ZoneDirective: Codable, Equatable {
    let zoneId: FrontZoneId
    let type: DirectiveType // "defend" 或 "attack"
    let parameters: DirectiveParameters
}
```

**防守指令 (`DefenseParameters`)**:
*   参数：`targetReserves: Int` (预备纵深), `stance: DefenseStance` (如 `holdLine`, `flexible`) 
*   **当前逻辑期待**：填线师（步兵）负责铺开防守，不要做复杂撤退。后续必定大改，但接口预留好。

**进攻指令 (`AttackParameters`)**:
*   参数：`targetTheaterId: TheaterId` (攻击的敌方战区大方向)
*   参数：`weightedRegions: [RegionId]` (重点攻击/主攻的省份区域)
*   参数：`intensity: AttackIntensity` (预留分级枚举：`infiltration`(渗透), `limitedCounter`(小规模反击), `allOut`(全面攻击)。目前只需实现基础的兵力全线压上)。

## 2. 自动编排器 (Directive Executor)
当 MockAI (或以后的 LLM) 丢出一个上述的 `ZoneDirective` 时，规则引擎应当自动向下翻译为微观单位运动。

*   新建 `WarCommandExecutor.swift` （或者类似的服务类）。
*   **解析 Defend**：拿到指令后，对该 `FrontZone` 的所有 `FRONT_UNIT` 执行自动平均调度算法。简单评估当前 segment 缺口，将无战事兵力调度填补防线缺口（仅做简单邻近填补，极其粗糙即可）。
*   **解析 Attack**：根据指定的 `targetTheaterId`，将具备攻击能力的主力师直接指派，命令他们向接触该战区的敌方 Region 推进（直接产生针对 hex 的 move & attack action 塞给底层的 `TurnManager` 或者行动队列）。注意，装甲/高机动师太复杂暂时在分配中过滤忽略，或者一视同仁当前线步兵扔着去平推。

---

## 3. MockAI 决策大脑
在 `WWIIHexV0/Agents/` 或对应层级，新建 `MockAICommander`，在这个版本中他扮演将领替身。
*   **视野机制约束**：获取当前战区的 `FrontLineState`。绝对不准偷看敌人的纵深，只获取**“与我方前线接壤的那一层敌军”**的总战力评价。
*   **算法判定**：
    *   通过对比 `Friendly Front Strength` vs `Visible Enemy Segment Strength`。
    *   如果**势均力敌或处于弱势**（例如战力劣于 120%）：生成并抛出 `DefenseParameters`。
    *   如果**具有明显优势**（例如超过 150%）：生成抛出 `AttackParameters`，并且自动把战力最集中的敌人 Region 放进 `weightedRegions` 权重数组里。
*   **工作流嵌入**：在游戏的回合流转 (GamePhase) 体系中，让非玩家阵营自动调用该策略大脑抛出 JSON Command，并送入 Executor 自动运行。

---

## 4. 全链路探针与验收要求 (Probe Tests)

在 `Tests/CommandSystemTests.swift` 或者对应模块下编写集成探针测试，必须覆盖以下情况：
1. **序列化测试**：使用手写的一段标准 JSON 文本（模拟 LLM 的粗糙输出），验证 `ZoneDirective` 可以将其反序列化。
2. **AI 决策判定探针**：MockAI 环境测试，在沙盒里给某战区摆 5 个己方步兵，对面摆 1 个步兵。断言 MockAI 输出的 `ZoneDirective.type == .attack`！如果我方只有 1 个步兵而对面有 5 个，必须断言输出为 `defend`。
3. **命令驱动底层探针**：向 `WarCommandExecutor` 发送一个 Attack Directive，在下一帧断言主力师的底层行为已被改写为了进行 `move/attack`。

> **寄语 Codex**：这套系统的核心使命是“为将来大模型的乱说乱答建立最稳固的格式保险与下层托底”。设计枚举和属性名字时，试想你是一个写 Python 或者写 Prompt 的人，尽量用容易理解的人类战术术语，而非干涩的数字序列编码。请开始。

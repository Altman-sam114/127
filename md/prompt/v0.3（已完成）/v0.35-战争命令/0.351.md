# v0.351 初级战争命令系统与 MockAI 驱动编排

## 目标

v0.351 在 v0.31-v0.33 的 `Region -> Theater/FrontZone -> FrontSegment -> Deployment` 链条上增加一层 LLM 友好的战争指令协议。

核心目标：

- LLM/MockAI 输出 `ZoneDirective` JSON。
- 规则侧把指令翻译成底层 `Command`。
- 底层仍走 `RuleEngine` 校验与执行。
- AI 只读取前线接触层敌军，不读取敌方纵深。

- `WWIIHexV0/Commands/WarDirective.swift`
- `WWIIHexV0/Commands/WarCommandExecutor.swift`

## 指令协议

根结构：

```swift
struct DirectiveEnvelope: Codable, Equatable {
    let schemaVersion: Int
    let issuerId: String
    let turn: Int
    let directives: [ZoneDirective]
}
```

战区指令：

```swift
struct ZoneDirective: Codable, Equatable {
    let zoneId: FrontZoneId
    let type: DirectiveType
    let parameters: DirectiveParameters
}
```

当前支持：

- `defend`
- `attack`

示例 JSON：

```json
{
  "zoneId": "germany_front",
  "type": "attack",
  "parameters": {
    "targetTheaterId": "france_front",
    "weightedRegions": ["sedan"],
    "intensity": "allOut"
  }
}
```

## 参数

防守：

```swift
struct DefenseParameters: Codable, Equatable {
    let targetReserves: Int
    let stance: DefenseStance
}
```

`DefenseStance`：

- `holdLine`
- `flexible`

进攻：

```swift
struct AttackParameters: Codable, Equatable {
    let targetTheaterId: TheaterId
    let weightedRegions: [RegionId]
    let intensity: AttackIntensity
}
```

`AttackIntensity`：

- `infiltration`
- `limitedCounter`
- `allOut`

## 执行链

完整链条：

```text
MockAICommander / future LLM
  -> DirectiveEnvelope JSON
  -> ZoneDirective
  -> WarCommandExecutor
  -> Command(move / attack / hold / allowRetreat)
  -> RuleEngine
  -> CommandResult
```

`WarCommandExecutor` 不直接改 `GameState`，只通过 `GameCommandHandling` 执行底层命令。

## Defend 行为

当前极简：

- 读取 `FrontZone.frontSegments`。
- 读取 `unitsFront` 和部分 `unitsDepth`。
- 找单位最少的 segment。
- 单位已在目标 region：`hold` 或 `allowRetreat`。
- 单位不在目标 region：生成 `move`。

不做：

- 复杂撤退。
- 路径搜索。
- 全图调度。

## Attack 行为

当前极简：

- 读取目标 `targetTheaterId` 对应的 `FrontZoneId`。
- 找与目标战区接触的 enemy region。
- 优先使用 `weightedRegions`。
- 若目标 region 有射程内敌军：生成 `attack`。
- 若目标 region 无敌军：生成 `move`。
- 所有命令仍经 `RuleEngine` 校验。

不做：

- 攻势箭头。
- 装甲特殊突破。
- 多步路径规划。
- 占领后重算战线的复杂联动。

## MockAICommander

输入：

- `GameState`
- `WarDeploymentState.frontZones`
- `FrontZone.frontSegments`
- 前线邻接 enemy region 内敌军

视野限制：

- 只统计与我方前线 segment 邻接的敌方 region。
- 不读取敌方纵深 region 的部队。

判断：

- `friendlyFrontStrength / visibleEnemyStrength >= 1.5`：attack。
- 否则 defend。

战力算法当前只做简化：

- 友军：`strength + attack`
- 敌军：`strength + defense`

## TurnManager 接入

`TurnManager` 保留旧 AgentOrder 管线。

只有满足以下条件才走 v0.351：

- 注入了 `directiveCommander`
- `state.warDeploymentState.frontZones` 非空
- 当前是 `germanAI` 阶段

App 启动默认给德军 AI 注入 `MockAICommander()`。

## Codable 调整

以下 ID 已改为单值字符串编码，方便 LLM JSON：

- `RegionId`
- `TheaterId`
- `FrontZoneId`

示例：

```json
"sedan"
```

而不是：

```json
{"rawValue": "sedan"}
```

## 测试

新增 `CommandSystemTests` 覆盖：

- `ZoneDirective` 从手写 JSON 解码。

## 当前限制

- 不做 UI。
- 不做聊天框。
- 不接真 LLM。
- 不做复杂 AI。
- 不做全局路径搜索。
- 不做完整 HOI4 战线计划。

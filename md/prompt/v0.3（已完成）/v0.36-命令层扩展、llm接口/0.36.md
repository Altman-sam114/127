# v0.36 战术指令体系重构与多将领 MockAI

## 背景与目标

v0.355 已完成：四层数据模型稳定（Hex/Region/Theater/FrontLine）、动态战区正确分离、占领判定统一、观察者模式可用、前线 overlay 可视化。

v0.36 的目标是**在现有 ZoneDirective 协议之上，引入可扩展的战术分类体系**，并将 MockAICommander 改造为**每战区一个将领 agent**，为后续接入真实 LLM 做好清晰接口。

核心要求：
1. 战术命令分层：**命令层（Command Category）→ 战术层（Tactic）**，目前实现进攻类-普通进攻、防御类-普通防御，设计上预留后续扩展。
2. 指令 JSON 扩展，包含战区、前线、命令类、战术名、命令目标等字段。
3. MockAICommander 拆分为多将领架构，每战区一个 `ZoneCommanderAgent`，概率分类器选择命令+战术。
4. 清晰的 LLM 替换接口：协议层和解码层分离，后续可零成本切换为真实 LLM 输出。
5. Probe 测试验证所有新接口和观察者模式行为。

## 二、战术分类体系设计

### 2.1 概念层级

```
CommandCategory（命令层）
    ├── offense（进攻类）
    │   └── standardAttack（普通进攻）   ← 当前实现
    │   [后续: blitzkrieg, fireSupression, infiltration, breakthrough]
    └── defense（防御类）
        └── holdPosition（普通防御）     ← 当前实现
        [后续: elasticDefense, depthDefense]
    [后续类别: retreat, support]
```

**本版只实现 `standardAttack` 和 `holdPosition`，但数据结构必须支持后续扩展。**

### 2.2 战术条件（TacticCondition）

每个战术有使用条件，版本 v0.36 先定义结构，`standardAttack` 和 `holdPosition` 条件设为空（始终可用）。

后续战术条件示例（**不要在 v0.36 中实现逻辑，只预留字段**）：
- `blitzkrieg`：需要将领有 `blitzkriegSkill`，需要有装甲单位
- `breakthrough`：需要兵力比 ≥ 2.0，且有连续前线段

### 3.2 扩展 `WarCommandExecutor.swift`

当前 `execute(_ directive:)` 按 `directive.parameters` 分支（defend/attack）。

v0.36：在同一位置**追加按 `tactic` 路由的逻辑**：

```swift
func execute(_ directive: ZoneDirective, in state: GameState) -> WarCommandExecutionResult {
    // 优先检查 v0.36 战术字段
    if let tactic = directive.tactic {
        return executeTactic(directive, tactic: tactic, in: state)
    }
    // 旧路径兜底（兼容 v0.355 以前的 directive）
    switch directive.parameters {
    case .defend(let parameters):
        return executeDefense(directive, parameters: parameters, in: state)
    case .attack(let parameters):
        return executeAttack(directive, parameters: parameters, in: state)
    }
}

private func executeTactic(
    _ directive: ZoneDirective,
    tactic: TacticName,
    in state: GameState
) -> WarCommandExecutionResult {
    switch tactic {
    case .standardAttack:
        // 与现有 executeAttack 完全相同的逻辑，直接复用
        guard case .attack(let parameters) = directive.parameters else {
            return emptyResult(directive: directive, state: state)
        }
        return executeAttack(directive, parameters: parameters, in: state)
    case .holdPosition:
        guard case .defend(let parameters) = directive.parameters else {
            return emptyResult(directive: directive, state: state)
        }
        return executeDefense(directive, parameters: parameters, in: state)
    // 后续新战术在此追加 case，不改旧代码
    }
}
```

`emptyResult` helper：返回空命令的 `WarCommandExecutionResult`（防御 fallback）。

#### 3.4.2 `ZoneCommanderAgent`（每战区一个将领 agent）

```swift
/// 单战区将领 agent
/// 职责：读取己方战区前线情况 → 概率分类 → 选择 CommandCategory + TacticName → 输出 ZoneDirective
struct ZoneCommanderAgent {
    let config: ZoneCommanderAgentConfig
    let conditionChecker: TacticConditionChecker

    init(config: ZoneCommanderAgentConfig, conditionChecker: TacticConditionChecker = .init()) {
        self.config = config
        self.conditionChecker = conditionChecker
    }

    func makeDirective(for zone: FrontZone, in state: GameState) -> ZoneDirective? {
        guard !zone.frontSegments.isEmpty else { return nil }

        let classification = classify(zone: zone, state: state)

        switch classification.category {
        case .offense:
            return makeOffenseDirective(tactic: classification.tactic, zone: zone, state: state)
        case .defense:
            return makeDefenseDirective(tactic: classification.tactic, zone: zone, state: state)
        }
    }
}
```

#### 3.4.3 概率分类器（BinaryTacticClassifier）

```swift
/// v0.36 简单二值分类器：基于兵力观测输出 attack/defend
/// 后续扩展：可替换为复杂评分函数，输出多类战术概率分布
struct BinaryTacticClassifier {
    let attackThreshold: Double

    /// 分类结果（命令层 + 战术名）
    struct Classification {
        let category: CommandCategory
        let tactic: TacticName
        let confidence: Double   // 分类置信度（供日志/LLM 上下文使用）
        let reason: String       // 可读原因
    }

    func classify(
        friendlyStrength: Int,
        visibleEnemyStrength: Int,
        hasContestedForwardPresence: Bool,
        hasStaticDefense: Bool,
        config: ZoneCommanderAgentConfig
    ) -> Classification {
        let ratio = visibleEnemyStrength == 0
            ? Double(friendlyStrength)
            : Double(friendlyStrength) / Double(visibleEnemyStrength)

        // 风格偏置
        let styleBoost: Double
        switch config.commandStyle {
        case .aggressive: styleBoost = 0.15
        case .balanced:   styleBoost = 0.0
        case .cautious:   styleBoost = -0.15
        }
        let adjustedRatio = ratio + styleBoost

        let shouldAttack = adjustedRatio >= attackThreshold
            || hasContestedForwardPresence
            || hasStaticDefense

        if shouldAttack {
            return Classification(
                category: .offense,
                tactic: .standardAttack,  // v0.36 只有一种进攻战术
                confidence: min(1.0, adjustedRatio / 2.0),
                reason: hasContestedForwardPresence ? "forward_presence" : "strength_ratio"
            )
        } else {
            return Classification(
                category: .defense,
                tactic: .holdPosition,    // v0.36 只有一种防御战术
                confidence: min(1.0, 1.0 / max(0.01, adjustedRatio)),
                reason: "outnumbered"
            )
        }
    }
}
```

**后续扩展点**：`BinaryTacticClassifier.classify` 的签名保持不变，但可以改为从外部注入复杂兵力因子（供给、地形、士气等），或整体替换为 LLM 输出的分类结果。

#### 3.4.4 将领池（TheaterCommanderPool）

```swift
/// 管理所有战区将领 agent 的池
/// TurnManager 只与 TheaterCommanderPool 交互，不再直接引用 MockAICommander
struct TheaterCommanderPool {
    /// 协议：未来可切换为 LLMZoneCommander
    protocol ZoneCommanderProviding {
        func makeDirective(for zone: FrontZone, in state: GameState) -> ZoneDirective?
        var config: ZoneCommanderAgentConfig { get }
    }

    private let commanders: [FrontZoneId: any ZoneCommanderProviding]

    init(commanders: [any ZoneCommanderProviding]) {
        self.commanders = Dictionary(
            uniqueKeysWithValues: commanders.map { ($0.config.assignedZoneId, $0) }
        )
    }

    func envelope(for faction: Faction, in state: GameState, issuerId: String = "theater_pool") -> DirectiveEnvelope {
        let directives = state.warDeploymentState.frontZones.values
            .filter { $0.faction == faction && !$0.frontSegments.isEmpty }
            .sorted { $0.id.rawValue < $1.id.rawValue }
            .compactMap { zone -> ZoneDirective? in
                commanders[zone.id]?.makeDirective(for: zone, in: state)
                    ?? ZoneCommanderAgent(config: defaultConfig(for: zone)).makeDirective(for: zone, in: state)
            }
        return DirectiveEnvelope(
            issuerId: issuerId,
            turn: state.turn,
            directives: directives
        )
    }

    private func defaultConfig(for zone: FrontZone) -> ZoneCommanderAgentConfig {
        ZoneCommanderAgentConfig(
            id: "default_\(zone.id.rawValue)",
            name: "Auto Commander",
            faction: zone.faction,
            assignedZoneId: zone.id,
            skills: [],
            commandStyle: .balanced
        )
    }
}
```

**兼容性**：保留 `MockAICommander` 结构不删（有 `// DEPRECATED` 不影响，但其内部逻辑现在应该委托给 `TheaterCommanderPool`/`ZoneCommanderAgent`），避免破坏已有测试。将 `TurnManager` 中调用 `MockAICommander.envelope(for:)` 的地方更新为调用 `TheaterCommanderPool.envelope(for:)`。

### 3.5 `AppContainer.swift` — 初始化将领池

`AppContainer` 在初始化/重置游戏时根据动态战区创建将领：

```swift
private func buildCommanderPool(state: GameState) -> TheaterCommanderPool {
    let zoneIds = state.warDeploymentState.frontZones.keys
    let configs = zoneIds.map { zoneId -> ZoneCommanderAgentConfig in
        let zone = state.warDeploymentState.frontZones[zoneId]!
        // 德军战区 → 进攻倾向命名
        let name: String
        let style: ZoneCommanderAgentConfig.CommandStyle
        if zone.faction == .germany {
            name = "German Commander (\(zoneId.rawValue))"
            style = .aggressive
        } else {
            name = "Allied Commander (\(zoneId.rawValue))"
            style = .balanced
        }
        return ZoneCommanderAgentConfig(
            id: "auto_\(zoneId.rawValue)",
            name: name,
            faction: zone.faction,
            assignedZoneId: zoneId,
            skills: [],
            commandStyle: style
        )
    }
    let agents = configs.map { ZoneCommanderAgent(config: $0) }
    return TheaterCommanderPool(commanders: agents)
}
```

**动态战区管控**：战区扩展后（`applyStrategicAdvance` 触发），新战区可能没有在 `AppContainer` 初始时注册的将领。`TheaterCommanderPool.envelope` 已处理这个情况（`defaultConfig` fallback），**无需每次扩展都重建整个 pool**。但如果需要，`AppContainer` 可以在 `endTurn` 后调用 `rebuildCommanderPoolIfNeeded`（按新战区 ID 补充缺口将领）。

# v0.34 MapEditor

## 目标

在 `MapEditor/` 下加入最小可运行地图编辑器骨架，用 SwiftUI 管理工具面板，用 SpriteKit 作为六角格交互视口。编辑器输出不走转换中间件，直接生成现有 `DataLoader` 可读取的 `ScenarioDefinition` 和 `RegionDataSet` JSON。

## 文件结构

- `MapEditor/MapEditorDocument.swift`：编辑器文档、Hex/Region/Theater/Unit 草稿模型、本地保存/读取。
- `MapEditor/MapEditorHexMath.swift`：轴向六角格坐标、屏幕点选 hit-test、hex 顶点计算。
- `MapEditor/MapEditorExporter.swift`：智能导出器，自动推导 region 邻接、region edges、道路边、representativeHex。
- `MapEditor/MapEditorViewModel.swift`：地图生命周期、绘制模式、region/theater 分配、导出状态。
- `MapEditor/MapEditorCanvasScene.swift`：SpriteKit 最小可视化六角格画布。
- `MapEditor/MapEditorView.swift`：SwiftUI 最小工具面板。
- `MapEditor/MapEditorMacApp.swift`：独立 macOS App 入口，使用 `MapEditorMac` target 运行。
- `MapEditor/MapEditorGameResourceBridge.swift`：读写 `WWIIHexV0/Data` 默认游戏资源。
- `WWIIHexV0/Tests/MapEditorOutputTests.swift`：编辑器输出到 `GameState` 的集成测试。

## 数据链路

```text
MapEditorDocument
  -> MapEditorExporter
  -> ScenarioDefinition JSON + RegionDataSet JSON
  -> DataLoader.loadGameState(scenarioName:regionName:)
  -> MapState.validateRegionGraph()
  -> TheaterState
  -> FrontLineState
  -> WarDeploymentState
```

## v0.34 兼容扩展

- `BaseTerrain` 增加 `hill`，并补 `terrain_rules.json`、地图颜色、Region inspector 显示名。
- `RegionNodeDefinition` 增加可选 `theaterId`，旧 JSON 不受影响。
- `DataLoader` 增加任意文件名加载入口和 MapEditor 输出专用 `loadGameState(scenarioName:regionName:)`。

## v0.341 macOS 独立编辑器

### Target

- 新增 Xcode target：`MapEditorMac`。
- 平台：macOS，默认窗口 `1440x900`，适合宽屏和全屏编辑。
- 入口文件：`MapEditor/MapEditorMacApp.swift`。
- 说明：MapEditor 现在作为独立 macOS App 运行，不接入 `WWIIHexV0App` 的 iOS 主入口。

### 交互

- 左侧使用 SwiftUI split panel 管理地图、模式、参数和文件操作。
- 右侧使用 SpriteKit `MapEditorCanvasScene` 渲染六角格。
- 支持鼠标按住拖拽连续涂色，不需要逐格点击。
- 支持滚轮/触控板缩放。
- 支持右键拖拽、中键拖拽、Option+左键拖拽平移。

### 模式

- Hex Painter：绘制 `plain/city/forest/mountain/hill`，可叠加 road、supply source、controller。
- Region Builder：新建/选择 region，用刷子加入 hex，用 erase 移除 region 归属。
- Theater Assignment：新建/选择 theater，点击 region 指派战区。
- Initial Unit Setup：选择 template、faction、HP、facing，像盖章一样把初始师部署到 hex；erase 可删除该 hex 上的初始部队。

### 游戏资源读写

默认工作流直接对接：

- `WWIIHexV0/Data/ardennes_v0_scenario.json`
- `WWIIHexV0/Data/ardennes_v02_regions.json`

按钮：

- `读取默认游戏资源`：从游戏 Data 目录读取默认地图和 region。
- `覆盖保存为游戏资源`：将当前编辑器文档导出并覆盖默认游戏资源。

导出后 iOS 主程序无需改 `DataLoader` 代码，即可继续加载默认地图。

## v0.342 中文化与显式编辑流

### 中文 UI

- App 左侧面板已全部改为中文。
- 模式显示为：`地块 / 省份 / 战区 / 部队`。
- 地形、阵营、部队模板、朝向、状态提示和资源按钮均使用中文。

### 分模式面板

- 左侧只显示当前模式相关控件，不再把 Hex、Region、Theater、Unit 控件混在同一面板。
- 切换模式会自动取消当前编辑会话，避免误操作。

### 显式编辑会话

所有模式统一使用：

- `添加`：进入添加状态，右侧地图点击或拖拽才会写入。
- `删除`：进入删除状态，右侧地图点击或拖拽删除当前模式对象。
- `完成`：提交当前模式的待选内容。
- `取消`：放弃当前待选内容。

说明：

- 地块模式：添加状态下直接覆盖 hex 地形/道路/补给/控制方；删除状态下重置该 hex，并移除该 hex 上的部队。
- 省份模式：添加状态下先选择多个 hex，完成后统一归入当前省份；删除状态下移除 hex 的省份归属。
- 战区模式：添加状态下选择多个 region，完成后统一归入当前战区；删除状态下清除 region 的战区归属。
- 部队模式：添加状态下选择多个 hex，完成后部署初始部队；删除状态下移除该 hex 上的初始部队。

### 分层显示

- 地块模式显示地形、道路、补给和控制方。
- 省份模式显示 region 分层颜色和当前待选 hex。
- 战区模式显示 theater 分层颜色和当前待选 region。
- 部队模式显示底层网格和单位标记。
- 非当前模式的数据不再用特殊颜色干扰当前编辑层。

## v0.343 视口稳定、稀疏扩图与快捷键

### 视口交互

- 右键/中键/Option+左键拖拽平移改为基于 view-space 指针增量计算，避免 camera 移动后 scene 坐标变化导致画面抖动。
- 鼠标滚轮缩放已接入 `scrollWheel`，触控板缩放已接入 `magnify`。
- 缩放使用鼠标所在 scene point 作为锚点，减少缩放时视口漂移。
- `MapEditorViewportMath` 提供纯数学探针，覆盖平移和锚点缩放计算。

### 稀疏地块扩展

- `MapEditorDocument.contains(_:)` 现在表示“该坐标实际存在 hex”，不再只是矩形宽高范围判断。
- 地块模式新增 `扩展地块` 动作：点击已有 hex 相邻空位时生成默认平原 hex。
- 禁止虚空造地块：新 hex 必须至少邻接一个已有 hex。
- 地块删除现在会真正删除 hex，并清理该 hex 上的初始部队；如果被删除 hex 所属 region 变空，会同步移除空 region 和 theater assignment。
- 导出 `ScenarioMapDefinition.isSparse` 会根据实际 hex 数量自动标记稀疏地图。

### 名称与 ID

- 省份/战区面板输入的是显示名称。
- 内部 ID 自动递增为 `region_1/region_2/...` 与 `theater_1/theater_2/...`。
- `准备新省份` / `准备新战区` 会清空当前选择，下一次添加时创建新的自动 ID 对象。

### 快捷键

- `N`：进入添加状态。
- `M`：完成当前编辑状态。

### 0.343 探针

## v0.344 交互修复、信息面板与底图层

### 快捷键

- macOS 版画布改用 `NSViewRepresentable + SKView` 承载 SpriteKit 场景。
- `SKView` 直接接收 `keyDown`，将 `N` 和 `M` 路由到 `MapEditorViewModel.handleShortcut(_:)`。
- 解决 SwiftUI `Button.keyboardShortcut` 在 SpriteKit 视图抢焦点后不稳定的问题。

### 滚轮缩放

- macOS 版 `SKView.scrollWheel` 直接调用 `MapEditorCanvasScene.zoomFromScroll(deltaY:anchor:)`。
- 垂直滚轮缩放地图，水平滚轮或 Shift+滚轮平移地图。
- 缩放仍以鼠标所在 scene point 为锚点。

### 信息面板

- 右键短按右侧地块会选中该 hex 并在左侧 `信息` 模块展示：
  - hex 坐标
  - 地形
  - 道路状态
  - region id / region 名称
  - theater id / theater 名称
- 信息面板可以直接编辑并保存 region 名称和 theater 名称。
- 右键拖动仍保留为平移地图；只有短按右键才触发查看信息。

### Region/Theater 颜色

- Region 和 Theater 改用固定高对比色板按 ID hash 取色。
- 省份层和战区层使用不同色板，避免相邻对象颜色过近。

### 底图层

- 新增 `MapEditorBackgroundImage`，仅属于编辑器文档，不写入游戏 Scenario/Region JSON。
- 支持从外部导入 `.png/.jpg/.jpeg`。
- 底图作为独立 SpriteKit 图片层绘制在 hex 网格下方，不影响地块、省份、战区、部队层。
- 左侧 `底图` 模块支持：
  - 导入/移除图片
  - 设置透明度
  - 设置缩放
  - 设置 X/Y 位置
- 该功能用于导入欧洲地图等参考图，再对着底图绘制省份。

## 当前限制

- 文件选择器、撤销/重做、批量框选、真实地图底图导入未实现。
- 战区 assignment 已进入导出 schema，但复杂战区编辑体验留给后续版本。
- 当前文件 I/O 先固定覆盖默认 Ardennes 测试图，后续再做多地图项目管理。
- 主游戏 `TheaterNode` 当前仍以 theater id 作为名称；编辑器内 theater name 已保存，但 RegionDataSet 尚未提供独立 theater name schema。

# v0.34 Phase 2: MapEditor macOS 端实装与核心体验进化 (Prompt for Codex)

> **前置说明**：此前你已经成功为 MapEditor 构筑了底层数据流、六角格数学与自动化导出验证，但仍然停留在没有实体界面应用和交互落地的“骨架阶段”。
> **这一阶段的目标是**：将 MapEditor 开发成一个独立的 **原生 macOS App Target**，并实打实地落地交互层和与游戏本体打通的文件层。加入部队部署模式并进行全链路跨端验证。

---

## 1. 独立 macOS 桌面端应用架构 (脱离 iOS 模拟器)
考虑到地图编辑器长期需要极高的性能和全屏宽屏的体验视角，在继续开发前，**不要将其接入原有的 iOS 主程序入口！**
*   **动作**：在 Xcode 工程中为 MapEditor 添加一个专门的 `macOS Target`。共用同一套 `Core` 代码。
*   **显示设定**：启用默认桌面级宽屏（Landscape），支持并建议全屏大屏运行。
*   **为什么这么做**：彻底避免 iOS Simulator 运行大片网格场景带来的性能开销耗电，纯跑原生 Mac M系芯片能够让你有如丝般顺滑的画图体验。

## 2. 操作与视觉体验重构 (方便交互)
抛弃目前“半残”的画板界面，为地图编辑器赋予现代制图工具级别的体验！
*   **无限缩放视口**：在 `MapEditorCanvasScene` 开放标准的摄像机（CameraNode）管理。必须支持用鼠标滚轮/触控板做到无限 **平移 (Pan) 和 缩放 (Zoom)**。
*   **顺滑涂色体验**：所有 Terrain（地形）、Region（省份划分）等模式下，不要逼人一个个点！必须实现**鼠标点住拖拽连笔涂色 (Drag-to-paint)** 功能。
*   **左区面板重调**：用 macOS 的 SplitView （或对应的 SwiftUI）将左侧编辑面板和画布进行分隔，左侧清晰展示参数调配。

## 3. 全新实盘模式：初期兵力部署 (Unit Planner Mode)
在上一版地块、省份、战区三个模式的基础上，追加游戏初局的单位布置系统。
*   **新增模式按钮**：「部队部署模式 (Initial Unit Setup)」。
*   **操作逻辑**：在左边面板选择部队模板，比如“装甲师”、“步兵师”、“重炮主力”。支持设置基础的暴露数值如兵力(HP)，归属阵营等参数，或者直接配置对应在模板池里的 `TemplateID`。
*   **画板印章**：配好数值后，就像盖章一样把师级图标或占位符点击部署到任意六角格上。部署好后应当能够被导出进 `ScenarioDefinition` JSON 文件的 `initialUnits` 列表。

## 4. 文件 I/O 系统：打通与 iOS 游戏的次元壁！
不要再仅在 Console 打印验证了，直接强挂钩真实工作流！
*   **直接读写目标目录**：将“Load / Save”操作对接向原工程中的 `WWIIHexV0/Data` 文件夹。你可以设定就读写覆盖一个指定的默认图（如 `ardennes_v0` 以及关联的 Region JSON），不需要做复杂的文件管理器。
*   **工作流体验**：在这个 Mac 独立编辑器里打开该 `Data` 里的 JSON，对地形/省份/部队位置做一波修改，直接点击「保存为游戏资源 (Overwrite)」。随后，在 Xcode 里立刻切到 iOS 项目启动手机模拟器，就能在手机游戏主页面立刻看到变动后的地形和部队分布。

## 5. 跨端最终验收标准
开发完成后，必须证明上述工作流是畅通且牢固的：
1. 请提供 Mac 端编辑器全屏开启的运行快照或详细操作测试日志。
2. 确保 `WWIIHexV0App`（iOS 主程序）的 `DataLoader` 可以零代码修改的情况下无缝加载经过 macOS 编辑器重构后的默认地图，并不造成任何 `MapEditorOutputTests` 及 RegionGraph 的失效报错。

**一切为了体验与所见即所得。请开始工作。**

# v6.8 Modern C2 UI Polish Progress

日期：2026-07-05

## 目标

按 `codex-v6.0-现代战争aiagent迁移总提示词.md` 的 v6.8 要求，推进发布级现代 C2 UI 首轮收口。范围限定为展示层和交互外观：不新增执行器，不改变 `Command` / `ZoneDirective` / `WarCommandExecutor` / `RuleEngine` 语义，不让 SwiftUI 或 SpriteKit 直接写战斗状态。

## 已实现

- 新增 `WWIIHexV0/UI/ModernCommandDesignTokens.swift`，集中管理 8pt 圆角、间距、44pt 最小触控区和 blue/red/green/neutral、sensor、fires、EW、sustainment、warning 色标。
- `HUDView` 从旧资源格子改为 C2 status strip，显示 turn、side、phase、victory、visible contacts、EW zones、ammo、air、supply risk 和 C2 queue。
- `ModernMissionPanelView` 改为 token 化面板，任务按钮继续只调用 `AppContainer`，并使用 `Label` / SF Symbols、44pt 最小高度和更清晰的 Supply / Contacts / Ammo 摘要。
- `NewGameButton` 使用统一 44pt 触控约束。
- `BoardScene` 新增 `drawModernC2Overlays`，在非 frontLine 图层绘制：
  - sensor coverage 青色 heatmap；
  - jammed sensor coverage 的 EW 色描边；
  - EW effect 紫色区域；
  - fire support result 火力环和 `F` / `!` 标记；
  - visible contact 稳定圆点，按 confidence 变色并显示估计类型缩写。
- `WWIIHexV0.xcodeproj/project.pbxproj` 已把新 Swift 文件加入 iOS 和 macOS source phase。

## 架构边界

- v6.8 overlay 只读 `GameState.operationalAwareness`、`GameState.fireSupportState`、`EconomyState` 和 `Division.supplyState`。
- 任务按钮仍通过 `AppContainer -> Command / ZoneDirective -> WarCommandExecutor / RuleEngine`。
- 没有新增 readiness / fuel / signature / electronicProtection 字段。
- 没有新增真实美术素材、第三方 UI 框架、网络请求、API key 或 LLM 依赖。

# v6.7 玩家现代指挥 UI、任务计划和人机协同进展记录

日期：2026-07-05

本轮目标来自 `codex-v6.0-现代战争aiagent迁移总提示词.md` 的 v6.7 段落：让玩家通过现代任务面板下达任务，同时保留微操能力，并保证玩家命令和 AI 命令共用后端 directive / command 管线。

## 已完成

- 新增 `WWIIHexV0/UI/ModernMissionPanelView.swift`。

- 任务面板展示：
  - selected formation
  - selected target sector / hex
  - formation supply summary
  - visible contact count
  - fire support ammo summary
- 任务面板提供 8 类现代任务：
  - Recon Area
  - UAV Orbit
  - Fire Mission
  - Air Support / SEAD
  - Assault Objective
  - Hold / Delay
  - Resupply / Repair
  - Jam / Counter-Drone
- `AppContainer` 新增玩家任务提交方法：
  - `orderModernReconArea()`
  - `orderModernUAVOrbit()`
  - `orderModernFireMission()`
  - `orderModernSuppressAirDefense()`
  - `orderModernElectronicWarfare()`
  - `orderModernResupplyRepair()`
  - `orderModernAssaultObjective()`
  - `orderModernHoldDelay()`
- `WWIIHexV0.xcodeproj/project.pbxproj` 已把 `ModernMissionPanelView.swift` 加入 iOS 和 macOS source phase。

## 任务落点

```text
Recon Area
  -> Command.recon
  -> CommandValidator
  -> VisibilityRules.performRecon

UAV Orbit
  -> Command.uavRecon
  -> CommandValidator
  -> FireSupportRules.validateUAVRecon / executeUAVRecon

Fire Mission
  -> Command.fireMission
  -> CommandValidator
  -> FireSupportRules.validateFireMission / executeFireMission

Air Support / SEAD
  -> Command.suppressAirDefense
  -> CommandValidator
  -> FireSupportRules.validateSuppressAirDefense / executeSuppressAirDefense

Jam / Counter-Drone
  -> Command.electronicWarfare
  -> CommandValidator
  -> VisibilityRules.applyElectronicWarfare

Resupply / Repair
  -> Command.resupply
  -> CommandValidator
  -> SupplyRules.applyResupplyRest

Assault Objective
  -> player ZoneDirective attack
  -> WarCommandExecutor
  -> RuleEngine

Hold / Delay
  -> player ZoneDirective defense
  -> WarCommandExecutor
  -> RuleEngine
```

## 架构边界

- SwiftUI View 不直接修改 `GameState`，只调用 `AppContainer` 闭包。
- 直接单位任务仍走 `submit(_ command:)`，由 `RuleEngine` 校验并执行。
- 宏观 Assault / Hold 仍复用既有玩家 `ZoneDirective` 路径，由 `WarCommandExecutor` 生成具体命令。
- observer mode 会禁用任务按钮。
- 任务拒绝原因继续通过 `lastCommandMessage`、interaction log 或 `WarDirectiveRecord` 展示。

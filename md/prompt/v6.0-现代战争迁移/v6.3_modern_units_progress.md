# v6.3 现代部队、装备、移动、战斗和后勤基础进度记录

日期：2026-07-05

## 目标

本轮推进 `codex-v6.0-现代战争aiagent迁移总提示词.md` 的 v6.3 阶段第一批实现：

- 默认灰潮剧本不再通过旧二战单位模板生成单位。
- 建立现代 `ComponentType` 与现代单位模板 catalog。
- 在不新增高风险 schema 字段的前提下，用现有 `components + strength + supplyState` 影响移动、战斗、补员和 UI 显示。
- 保持玩家、AI、MockAI 和后续聊天命令仍经 `ZoneDirective / Command -> WarCommandExecutor -> RuleEngine`。

## 已落地

数据：

- 新增 `WWIIHexV0/Data/modern_unit_templates.json`。
- `grey_tide_2030_scenario.json` 初始单位改用 `recon_screen`、`mechanized_task_force`、`fires_battery`、`security_detachment`、`air_defense_detachment` 等现代模板。
- `WWIIHexV0.xcodeproj/project.pbxproj` 已把 `modern_unit_templates.json` 加入 iOS / macOS resource phase。

源码：

- `ComponentType` 新增 `armor`、`mechanizedInfantry`、`lightInfantry`、`recon`、`rocketArtillery`、`airDefense`、`engineer`、`logistics`、`uav`、`loiteringMunition`、`specialForces`、`electronicWarfare`。
- `DataLoader.loadUnitTemplates()` 优先读取 `modern_unit_templates.json`；`loadArdennesDataSet()` 固定读取旧 `unit_templates.json`，保护旧数据集和历史 fixture。
- `MovementRules` 加入装甲/机械化复杂地形惩罚，工程和后勤支援可缓解部分移动成本。
- `CombatRules` 加入机械化道路攻击、工程突破、防空/EW 抑制无人系统、轻步兵山地防御等首版 modifier。
- `Division.effectiveStats` 中后勤组件会缓解低补给和包围衰减。
- `EconomyRules` 补员成本按装甲、机动、火力、防空/无人、工程/后勤组件权重计算。
- `WarCommandExecutor`、`ZoneCommanderAgent` 的机动判定已识别现代机械化 family。
- `TacticName.displayName` 和 `WarDirectiveRecord.tacticDisplayName` 提供现代战术展示名，保留 raw value 兼容 AI JSON。
- `UnitInspectorView`、`UnitTooltipView`、`UnitNode` 使用现代 component display code / family helper。
- MapEditor 默认单位模板和模板 Picker 改为现代任务编组、侦察、火力、防空、工程、后勤等语义。

## 仍未完成

- ISR / ContactTrack / SensorCoverage / EWEffect 尚未建模。
- FireMission / AirTasking / 精确火力命令尚未独立接入 `Command` / `ZoneDirective`。
- readiness / ammo / fuel / signature / electronicProtection 尚未作为独立字段落库。
- 默认灰潮地图仍是 60-hex 种子，不是 100-220 hex 发布地图。
- `GamePhase.germanAI/alliedPlayer`、`Division` 源码名、旧 `ProductionKind` case、旧阿登 JSON 和历史测试 fixture 继续保留兼容。

## 验收关注

- 默认新局应继续加载 `grey_tide_2030`。
- 灰潮单位应从现代模板解析出现代组件，不再依赖旧 `unit_templates.json`。
- 旧阿登数据集仍应通过 legacy 模板校验。
- 玩家和 AI 的移动、攻击、防守、补给仍只能通过 `WarCommandExecutor -> RuleEngine` 执行。
- 本机只运行 `md/test/test.md` 允许的轻量检查；Xcode build/test、模拟器和 Probe/Smoke/Full 继续交给 GitHub Actions 或人工授权。

# v6.4 ISR / ContactTrack / EW Progress

日期：2026-07-05

本轮目标：完成 v6.4 第一版 ISR、战争迷雾、ContactTrack 和电子战基础。范围只覆盖发现、确认、干扰和可见摘要隔离；不实现 v6.5 的 FireMission、AirTasking、防空压制或无人打击闭环。

> 历史说明：本文件记录 v6.4 当轮状态。后续 v6.5+ 已接入 FireMission / AirTasking 抽象，v6.10 已补 cloud Probe、Playtest 状态链路和 restricted / civilian fire ROE 证据；当前检查规则以 `md/test/test.md` 为准，本文件下方本机轻量检查计划不代表当前默认执行规范。

## 已落地

- 新增 `OperationalAwarenessState`，包含 `ContactTrack`、`SensorCoverage` 和 `EWEffect`。
- `GameState` 新增 `operationalAwareness`，旧存档 decode 缺字段时 fallback `.empty`。
- 新增 `VisibilityRules`：
  - 从友军 vision、recon、UAV、防空、EW 组件生成 sensor coverage。
  - 覆盖内敌军生成或刷新 contact。
  - EW effect 降低目标 side 的 sensor quality。
  - 回合推进时 contact 老化、降级、消失，EW remainingTurns 递减。
  - 提供 `targetQuality` 和 `contactStrengthEstimate` 给后续火力任务复用。
- `Command` 新增 `recon` 和 `electronicWarfare`，经 `CommandValidator -> CommandExecutor -> VisibilityRules` 执行，写入 `intelligence` / `electronicWarfare` 日志。
- `StrategicStateBootstrapper` 在 bootstrap 和 runtime refresh 后刷新作战感知状态。
- AI 摘要从真实敌军列表收窄为 visible contacts：
  - `AgentContext.contactSummaries`
  - `AgentPromptBuilder` Visible contacts
  - `ZoneCommanderAgent` / `MarshalBattlefieldSummarizer` / `MockAICommander` contact strength estimate
- `WarCommandExecutor` 的直接攻击目标解析改为 contact-gated：只把 medium+ contact 的内部 `linkedDivisionId` 解析成真实目标。
- 普通 UI 视角不再显示敌军 `Division` 兵牌；Region inspector 显示 contact 类型、可信度、来源和年龄。Observer mode 保留调试全显。
- Xcode project 已把 `OperationalAwarenessState.swift` 和 `VisibilityRules.swift` 加入 iOS、macOS 与 MapEditor 相关 sources。

## 架构边界

- Hex 仍是战术权威；contact 只是可见摘要，不改变真实单位坐标和占领。
- `linkedDivisionId` 只供规则内部解析，不进入 AI prompt 或 Region inspector。
- `recon` 不造成伤害；`electronicWarfare` 只影响感知质量，不直接杀伤。
- AI / UI 不再默认读取真实敌军 `Division` 列表；执行层只有在已有足够 contact 时才能把 contact 解析为真实目标。

## 仍未完成

- FireMission / AirTasking / 精确火力命令未建模。
- 防空压制、无人打击、弹药、燃料、readiness、signature、electronicProtection 未独立落库。
- contact 地图 overlay 仍是首版：Region inspector 和攻击高亮已接入，完整独立图层留后续 UI 轮次。
- legacy AgentOrder JSON schema 仍不能直接下发 recon / EW；主战争 AI 路径继续走 Marshal / ZoneDirective。

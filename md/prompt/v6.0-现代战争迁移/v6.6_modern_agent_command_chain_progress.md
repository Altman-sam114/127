# v6.6 现代 AI Agent 指挥链进展记录

日期：2026-07-05

本轮目标来自 `codex-v6.0-现代战争aiagent迁移总提示词.md` 的 v6.6 段落：构建现代作战味道的 AI Agent 层级，让 Agent 输出可审计、可回放、可校验的结构化 directive，并继续保证最终行动不绕过 `WarCommandExecutor` 和 `RuleEngine`。

## 已完成

- 新增 `WWIIHexV0/Agents/ModernCommandChain.swift`。
- 定义 Codable 指挥链 DTO：
  - `StrategicConstraintEnvelope`
  - `JointCommandPlan`
  - `ModernSubDirective`
  - `ModernCommandChainPlan`
- 定义现代指挥角色：
  - `nationalCommand`
  - `jointCommand`
  - `chiefOfStaff`
  - `isrCoordinator`
  - `firesCoordinator`
  - `airTasking`
  - `ewCoordinator`

  - `brigadeCommander`

  - `setROE`
  - `theaterObjective`
  - `deconflict`
  - `reconArea`
  - `confirmContact`
  - `fireMission`
  - `suppressAirDefense`

  - `assault`
  - `hold`

- 新增 `ModernCommandChainDecoder`，校验：
  - 顶层 `schemaVersion`、`issuerId`、`turn`、`faction`
  - 嵌套 `StrategicConstraintEnvelope` / `JointCommandPlan` 的 schema、issuer、turn、faction、role
  - sub-directive 的 role / missionType 合法组合
  - `zoneId` 是否存在且属于当前 faction
  - `regionId` 是否存在
  - `contactId` 是否存在且对当前 faction 可见
- 新增 deterministic `ModernCommandChainOrchestrator`，根据 `MarshalBattlefieldSummary`、`TheaterDirectiveEnvelope` 和 `GameState` 生成 fenced JSON。
- `MarshalAgent` 在 TheaterDirective 成功 decode 后生成并复核 Modern Command Chain JSON；失败只追加 diagnostics，不执行半成品。
- `TurnManager` 将以下三段写入 `AgentDecisionRecord.rawJSON`：
  - TheaterDirective JSON
  - Modern Command Chain JSON
  - Compiled ZoneDirective JSON
- `parsedIntent` 附加 `ModernCommandChainPlan.summary`，让 AI 面板能看到现代指挥链摘要。
- `WWIIHexV0.xcodeproj/project.pbxproj` 已把 `ModernCommandChain.swift` 加入 iOS、MapEditorMac 和 macOS source phase。

## 架构边界

- v6.6 的现代指挥链当前是 advisory 层，不直接发 `Command`，不直接改 `GameState`。
- 最终行动仍走：

```text
TheaterDirectiveCompiler
  -> ZoneDirective
  -> WarCommandExecutor
  -> RuleEngine
```

- 外部模型输出仍必须是 fenced JSON 或纯 JSON，并由 decoder 校验；校验失败不能执行半成品。
- 本轮没有引入 API key、模型路径、网络调用或真实本地 LLM 多 Agent 并发。

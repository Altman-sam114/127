# v6.1 作战方、ROE 与中立兼容进展

> 本记录对应 v6.1 的第一批实现。目标是让现代作战方和中立语义进入底层兼容层，而不是一次性完成全项目多方战争、通用 phase 和现代默认剧本。

## 1. 本轮目标

- 在不破坏旧 `germany/allies` JSON 的前提下，让数据可以表达 `blueForce`、`redForce`、`greenForce`、`neutral`。
- 建立最小 ROE / hostile helper，逐步替代 `Faction.opponent`。
- 修复 `RegionDataSet.toRegions()` 中 owner/controller 缺失时 fallback 到 `.allies` 的历史债。
- 保持 `Command` / `ZoneDirective -> WarCommandExecutor -> RuleEngine` 权威链不变。

## 2. 已落地

### 2.1 Faction 兼容扩展

`Faction` 当前支持：

- 旧兼容：`germany`、`allies`
- 现代作战方：`blueForce`、`redForce`、`greenForce`、`neutral`

新增 `OperationalSideAlignment`：

- `blue`
- `red`
- `green`
- `neutral`

新增 helper：

- `Faction.dataValue(_:)`：用于数据加载时解析旧值和现代 alias。
- `alignment`：把旧兼容阵营和现代作战方归到 blue/red/green/neutral。
- `usesPlayerCommandPhase` / `usesAICommandPhase`：先把旧 `alliedPlayer/germanAI` phase 映射到 blue/red。
- `isHostile(to:)` / `defaultROEStatus(toward:)`：最小 ROE 判断。

支持的数据值示例：

```text
allies -> allies
germany -> germany
blueForce / blue_force / power_blue -> blueForce
redForce / red_force / power_red -> redForce
greenForce / power_green -> greenForce
neutral / civilian -> neutral
```

### 2.2 Diplomacy / ROE

- `DiplomaticStatus` 新增 `restricted`。
- 初始外交关系不再只按“同 faction allied / 不同 faction atWar”处理，而是通过 `defaultROEStatus(toward:)` 生成。
- `DiplomacyState.initial(from:)` 使用 `Faction.dataValue` 解析现代数据值。
- `DiplomacyState.initial(for:)` 可为 red / blue / green / neutral 生成初始 profile 和 bloc。

### 2.3 Region 中立 fallback

`RegionDataSet.toRegions()` 现在规则：

```text
controller 缺省 -> owner
owner/controller 都缺省 -> neutral
```

这修复了旧逻辑中 `nil owner/controller` 被错误算给 `.allies` 的风险。

### 2.4 已替换的二元敌我判断

以下位置已从 `.opponent` 或二元判断改为 ROE hostile helper：

- `SupplyRules`
- `RegionSupplyRules`
- `RegionCombatRules`
- `AgentContexts`
- `WarCommandExecutor` 的目标排序
- `FrontLineManager` 的前线对手推导
- `ZoneCommanderAgent` 的 lost objective 摘要

### 2.5 Phase / AI 兼容

- `CommandValidator.phaseAllowsCommands` 改用 `usesAICommandPhase` / `usesPlayerCommandPhase`。
- `CommandExecutor.executeEndTurn` 支持 `redForce -> blueForce -> redForce` 的现代两方回合循环。
- `AppContainer` 和 `TurnManager` 的 AI 判断可识别 red / blue alignment。
- `RulerAgent`、`MarshalAgentConfig.automatic`、`TerrainStyle`、`MapEditor` 的 `Faction` switch 已适配新增 case。

### 2.6 经济初始化兼容

- `EconomyRules.makeInitialState` 在没有传入阵营时只回退到旧双阵营 `legacyBelligerents`，不因 `Faction.allCases` 扩容而默认给 green / neutral 建账本。
- `EconomyRules.bootstrapIfNeeded` 按存档实际单位阵营加旧双阵营 fallback 初始化账本；现代剧本若显式放入 red / blue / green / neutral 单位，仍会按实际阵营建账本。

## 3. 未完成

- 仍保留 `Faction.opponent` 作为旧接口兼容；当前主路径 fallback 已不再调用二元 opponent，源码调用残留应只限兼容属性本身和历史文档。
- `GamePhase` raw value 仍是 `germanAI/alliedPlayer/resolution`，只是显示名和 helper 通用化。
- 默认数据已在 v6.2 后切到 `grey_tide_2030`，v6.10 扩展到发布候选规模；运行时验证仍待云端和人工授权补测。
- MapEditor 现代默认资源桥已在后续版本切到灰潮数据；旧 `allies/germany` 只保留为兼容。
- CommandValidator / FireMission 已有抽象 ROE 与目标质量检查；restricted / civilian region 的完整深水火力规则仍待后续版本细化。

## 4. 验收边界

本轮只能证明：

- 现代作战方值可以进入现有 Swift 模型。
- 中立 region fallback 不再落到 `.allies`。
- 一批核心敌我判断已从二元 opponent 迁移到 ROE helper。

本轮不能证明：

- `grey_tide_2030` 可运行。
- 多方 AI 完整行动正确。
- 现代火力 / EW / ISR 已受 ROE 约束。
- UI 视觉已发布级。

这些仍要在 v6.2-v6.10 继续实现和验证。

# v6.10 发布候选准备进度

日期：2026-07-05

本文件记录 `codex-v6.0-现代战争aiagent迁移总提示词.md` 的 v6.10 收口结果。当前结论是：发布候选代码和文档已准备，云端 build 验证待本轮 push 后核对；运行时启动、UI 点击、截图、10-20 回合观察和性能体感仍未获人工授权，不能声明正式发布或发布级运行时已验证。

## 1. 本轮目标

- 补齐 v6.0-v6.10 的发布候选说明、残留扫描和人工授权重验证清单。
- 消除本轮确认的玩家可见主应用二战命名残留。
- 保持 hex / region / theater / front / deploy / command 规则权威不变。
- 不做 target、module、bundle id、源码类型名和旧 fallback 数据的全局大重命名。

## 3. 本轮残留扫描结论

本轮扫描范围：

```sh
rg -n "INFOPLIST_KEY_CFBundleDisplayName|PRODUCT_NAME|PRODUCT_BUNDLE_IDENTIFIER|MARKETING_VERSION|CURRENT_PROJECT_VERSION" WWIIHexV0.xcodeproj/project.pbxproj
rg -n "WWII|Ardennes|Bastogne|Panzer|Guderian|Germany|Allies|Division|German|Allied" WWIIHexV0/App WWIIHexV0/UI WWIIHexV0/SpriteKit WWIIHexV0/Data README.md md/flow/flow.md update_log.md
rg -n "displayName|tacticDisplayName|operationalDisplayName|localizedName|scenarioDisplayName|CFBundleDisplayName|SUP A|SUP G|infantryDivision|panzerDivision|motorizedDivision" WWIIHexV0/App WWIIHexV0/UI WWIIHexV0/SpriteKit WWIIHexV0/Core WWIIHexV0/Data
```

处理结果：

- `WWIIHexV0.xcodeproj/project.pbxproj` 中 iOS / macOS 主游戏 `INFOPLIST_KEY_CFBundleDisplayName` 原为 `WWII Hex V0` / `WWII Hex V1.1 Mac`，已改为 `Modern Command Agent`。
- `WWIIHexV0/Assets.xcassets/AppIcon.appiconset` 已加入 iOS / macOS 主游戏 resources，并提供 iPhone、iPad、macOS 和 iOS marketing 所需尺寸；图标为抽象现代 C2 网络标识，不使用现实国家旗帜或敏感战场符号。
- `ModernPlaytestPanelView` 已把 Player Side、Opposition 和 Control Mode 放在 Playtest status 区，避免玩家只看到旧兼容 faction 或不清楚自己当前扮演方。
- `WWIIHexV0/SpriteKit/HexNode.swift` 中补给源标签原为 `SUP A` / `SUP G`，已改为按阵营 alignment 派生的 `SUP B` / `SUP R`，旧 `.allies` / `.germany` fallback 也显示为现代 Blue / Red。
- `EconomyState.ProductionKind` 仍保留 `infantryDivision` / `panzerDivision` / `motorizedDivision` raw case 名，但 `displayName` 已是 `Light Infantry Task Force`、`Armored Task Force`、`Mechanized Task Force`，属于源码兼容残留。
- `unit_templates.json`、`ardennes_*`、`general_agents.json`、`generals.json`、`DataLoader.loadArdennesDataSet()` 和 `Legacy Fallback Scenario` 属于旧数据集 / fallback / 历史兼容残留，不是默认现代剧本主路径。
- `README.md`、`md/flow/flow.md`、`update_log.md` 中仍记录 WWIIHexV0、Ardennes、Guderian、Division 等历史名，用于迁移背景和兼容边界说明，不应被当作主游戏 UI 文案。
- target / module / bundle id 仍保留 `WWIIHexV0` / `WWIIHexV0Mac` / `com.example.WWIIHexV0*`，本轮不改，原因是全局改名会牵涉 Xcode target、scheme、测试、CI、source membership 和历史 imports，超出 v6.10 发布候选收口范围。

## 4. 发布前需人工授权的重验证

未获授权前不得主动在本机执行：

- Xcode build。
- iOS Simulator 或真机启动。
- macOS target 启动。
- 10-20 回合 observer 模式。
- 基础 UI 点击烟测。
- SpriteKit 截图或人工视觉检查。
- 性能体感检查。

当前可声明的范围：

- 本轮只做轻量语法、格式、JSON、project 文件和云端 artifact 检查。
- 若本轮 push 后 GitHub Actions 通过，只能声明“发布候选代码和文档已准备，云端 build 通过”；App 图标资源已静态存在，但 Dock / SpringBoard 实机显示仍需人工授权启动验证，不能声明“已发布”或“运行时发布级已验证”。

## 5. 下一步建议

1. 人工授权一次发布前运行时验证：iOS / macOS 启动、UI 点击、SpriteKit 截图、10-20 回合 observer 和性能体感。
2. 若运行时验证失败，继续在 `main` 上追加 v6.10.x 修复 commit，并重新下载 artifact 核对。
3. 若运行时验证通过，再考虑 v6.11 做完整红/蓝新局选择器，以及 target / bundle id / scheme 的正式产品命名迁移，避免在发布候选阶段引入工程级重命名风险。

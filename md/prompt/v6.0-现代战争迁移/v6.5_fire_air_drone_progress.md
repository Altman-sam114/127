# v6.5 精确火力、空地协同、无人系统和防空抽象进展记录

日期：2026-07-05

本轮目标来自 `codex-v6.0-现代战争aiagent迁移总提示词.md` 的 v6.5 阶段：建立“侦察 -> 确认目标 -> 防空压制 -> 火力打击 -> 地面推进”的现代作战首版闭环。范围控制为抽象任务模型，不做复杂实时空战、真实武器数据库、完整空优系统或 fuel/readiness/signature 独立落库。

## 已完成

- 新增 `FireSupportState`：
  - `ammoBudgetBySide`
  - `cooldownsByAsset`
  - `scheduledMissions`
  - `lastMissionResults`
  - `AirTaskingState`
- 新增 `FireMission` / `FireMissionResult` / `MunitionClass` / `FireMissionTarget` / `FireRiskFlag` 模型。
- 新增 `AirTaskingState`、`AirSortie`、`AirDefenseThreat`、`AirDefenseSuppression` 抽象模型。
- `GameState` 增加 `fireSupportState`，旧存档缺字段时回退 `.initial`。
- `Command` 增加：
  - `fireMission(issuerId:target:munitionClass:)`
  - `uavRecon(divisionId:target:)`
  - `suppressAirDefense(divisionId:target:)`
- `CommandValidator` 统一校验新命令的阶段、阵营、行动权、目标、射程、source asset、目标质量、弹药、冷却、防空威胁和友军邻近风险。
- `CommandExecutor` 通过 `FireSupportRules` 执行火力 / UAV / 防空压制任务，并在回合推进时衰减 cooldown、sortie 和 suppression。
- `VisibilityRules.performUAVRecon` 支持强制 UAV source，并根据 AD/EW 风险降低 contact confidence。
- `FireSupportRules` 执行首版效果：
  - fire mission 只接受本方可见 medium+ contact 或含 medium+ contact 的 hex / region。
  - 命中时造成有限 strength damage，必要时触发 retreat / destroyed。
  - high AD / EW 会降低效果或导致任务失败。
  - `suppressAirDefense` 写入持续若干回合的 AD suppression，降低后续空中/无人任务风险。
- `WarCommandExecutor` 的 `fireCoverage` 战术会先尝试生成 contact-gated `fireMission`，再继续原有地面 attack / hold fallback。
- 新增 `fireSupport` 日志分类，战报显示 success / degraded / failed / suppressed 和风险原因。
- 同步更新：
  - `README.md`
  - `md/flow/flow.md`
  - `md/flow/flowchart.md`
  - `update_log.md`

## 架构边界

- FireMission 不占领 hex，不改 `hexToTheater` / `hexToFrontZone`，只影响 damage、retreat、destroyed、contact/fire observation 和日志。
- 新命令仍必须走 `Command -> CommandValidator -> CommandExecutor -> RuleEngine`，禁止直接改 `GameState`。
- `linkedDivisionId` 仍只作为规则内部解析字段，AI/UI 摘要默认不暴露真实敌军单位 id。
- `fireCoverage` 只做首版自动火力任务；独立 FiresCoordinator / ISRCoordinator Agent 留给 v6.6+。

## 未完成 / 风险

- 未做真实武器数据库、弹种库存分层、fuel、readiness、signature、electronicProtection 独立字段。
- Air superiority 仍是 `AirTaskingState` 中的抽象占位，未实现 CAP、截击、机场出动率或持续空域争夺。
- 火力范围 / 空中任务 overlay 未做，当前主要通过 EventLog 和 mission result 状态可见。
- 本轮按 `md/test/test.md` 不主动跑本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Full；重验证以 GitHub Actions artifact 为准。

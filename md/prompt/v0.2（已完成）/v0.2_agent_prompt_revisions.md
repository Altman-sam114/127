# v0.2 Agent 3-6 提示词改动说明（叠加方向）

## 背景

原 v0.2 提示词（`md/v0.1～1.0提示词/v0.2.md`）是**旧版纯替换方案**：把 hex 规则迁成 region 权威，删 HexCoord。

**新方向已定：叠加，不替换。** 三层架构：

```
[战略层] Province（省 / region）—— Agent 1/2 已落地模型+数据
  - 占领归属 / 资源 / 补给大方向 / AI 战略判断单位
        ↓ 映射（hexToRegion）
[战术层] Hex Grid（局部激活，六角格）—— 现有 hex 规则全保留
  - 战斗 / 移动细节 / 局部地形遮挡 / 包围侧翼撤退 / 战术 AI
        ↓
[视觉层] Map Rendering（SpriteKit）—— 现有渲染保留
  - hex 绘制 / 省份边界 overlay / 战争迷雾
```

**核心原则：hex 规则不迁、不删、不替换。省级规则是新增并行层。**

## Agent 3 提示词改动（规则迁移 → 省级规则并行）

### 原提示词要改的措辞

| 原措辞（删/改） | 新措辞 |
|---|---|
| "把规则从 hex 迁移到 region" | "新增省级规则，与现有 hex 规则并行运行" |
| "移动：RegionId → RegionId（替换 HexCoord）" | "新增 RegionMovementRules（省级移动），现有 MovementRules（hex）保留不动" |
| "攻击：region graph distance（替换 hex distance）" | "新增 RegionCombatRules（省级攻击），现有 CombatRules 保留" |
| "补给：region pathfinding（替换 hex）" | "新增 RegionSupplyRules（省级补给聚合），现有 SupplyRules 保留" |
| "HexCoord 兼容包装：HexCoord → RegionId → RegionRules" | 删。不包装、不转换。两套规则独立，各自输入不同坐标层 |
| "Division.location: RegionId" | 删。Division.coord 保持 HexCoord |

```

Rules/CombatRules.swift        # hex 战斗，保留
Rules/SupplyRules.swift        # hex 补给，保留
Rules/CommandExecutor.swift    # hex 命令执行，保留
```

### 完成标准改动

| 原 | 新 |
|---|---|
| "移动/攻击/占领/补给都基于 RegionId 工作" | "省级规则能跑通，hex 规则同时仍跑通" |
